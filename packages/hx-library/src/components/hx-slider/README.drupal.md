# hx-slider — Drupal Integration Guide

## Overview

`hx-slider` is a form-associated range slider that participates in native HTML forms via the ElementInternals API. It renders a track, thumb, optional tick marks, and label. In Drupal, it can be used inside any `<form>` element, including those generated by Drupal's Form API.

## Twig Template

Use the provided `hx-slider.twig` template. Pass variables for the slider properties.

### Template Variables

| Variable         | Type                      | Default    | Description                                               |
|------------------|---------------------------|------------|-----------------------------------------------------------|
| `name`           | string                    | required   | Form field name submitted with the form                   |
| `min`            | number                    | `0`        | Minimum allowed value                                     |
| `max`            | number                    | `100`      | Maximum allowed value                                     |
| `value`          | number                    | `0`        | Initial/default value (restored on form reset)            |
| `step`           | number                    | `1`        | Stepping interval between values                          |
| `label`          | string                    |            | Visible label text                                        |
| `help_text`      | string                    |            | Help/hint text shown below the slider                     |
| `size`           | `'sm'` \| `'md'` \| `'lg'`| `'md'`    | Size variant                                              |
| `show_value`     | boolean                   | false      | Show the current numeric value next to the label          |
| `show_ticks`     | boolean                   | false      | Render tick marks at each step interval                   |
| `aria_valuetext` | string                    |            | Human-readable value label for screen readers             |
| `disabled`       | boolean                   | false      | Prevent interaction                                       |

## Example Usage

```twig
{# Basic slider inside a Drupal form #}
<hx-slider
  name="dosage_level"
  min="0"
  max="10"
  value="5"
  label="Dosage Level"
></hx-slider>
```

```twig
{# Pain scale with contextual value labels #}
<hx-slider
  name="pain_score"
  min="0"
  max="10"
  value="{{ patient.pain_score|default(0) }}"
  step="1"
  label="Pain Level (0–10)"
  show-value
  show-ticks
></hx-slider>
```

```twig
{# Include via template #}
{% include 'hx-slider.twig' with {
  name: 'pain_score',
  min: 0,
  max: 10,
  value: patient.pain_score|default(0),
  label: 'Pain Level',
  show_value: true,
  aria_valuetext: pain_level_labels[patient.pain_score|default(0)],
} %}
```

## Form Integration

### Drupal Form API Render Element

`hx-slider` can be used as a custom render element or embedded directly in a Twig template alongside native form fields:

```twig
<form method="post" action="{{ form_action }}">
  {{ form.hidden_fields }}

  <hx-slider
    name="{{ element['#name'] }}"
    min="{{ element['#min']|default(0) }}"
    max="{{ element['#max']|default(100) }}"
    value="{{ element['#default_value']|default(0) }}"
    label="{{ element['#title'] }}"
  ></hx-slider>

  <button type="submit">Submit</button>
</form>
```

### Form Reset Semantics

`hx-slider` follows native HTML form reset behavior: when the form is reset, the slider restores to the value set in the `value` attribute at render time — not to `min`. This matches the behavior of `<input type="range">`.

```twig
{#
  A slider with value="5" will reset to 5, not to min="0",
  when the user clicks a <button type="reset"> or the form is programmatically reset.
#}
<hx-slider name="threshold" min="0" max="100" value="5"></hx-slider>
```

### FormData Submission

The slider value is included in form `FormData` under the `name` attribute:

```js
const form = document.querySelector('form');
form.addEventListener('submit', (e) => {
  e.preventDefault();
  const data = new FormData(form);
  console.log(data.get('pain_score')); // '7' (string)
});
```

## Drupal Behaviors Integration

Listen to slider events via Drupal behaviors to update dependent UI:

```js
(function (Drupal, once) {
  Drupal.behaviors.hxSlider = {
    attach(context) {
      once('hx-slider-init', 'hx-slider[name]', context).forEach((slider) => {
        // hx-input fires continuously while dragging
        slider.addEventListener('hx-input', (e) => {
          const { value } = e.detail;
          // Update a related display or trigger a live preview
        });

        // hx-change fires when the user releases the thumb
        slider.addEventListener('hx-change', (e) => {
          const { value } = e.detail;
          // Persist value, trigger AJAX, or update form state
        });
      });
    },
  };
})(Drupal, once);
```

## Progressive Enhancement

If JavaScript fails to load or is blocked, the `hx-slider` host element renders as an empty custom element with no visible input. For forms where the slider value is critical, provide a native `<input type="range">` fallback inside the element using the default slot:

```twig
<hx-slider name="dosage" min="0" max="10" value="5" label="Dosage">
  {# Native fallback — visible only when hx-slider is not upgraded #}
  <noscript>
    <input type="range" name="dosage" min="0" max="10" value="5">
  </noscript>
</hx-slider>
```

## Attribute Notes

| Attribute      | Drupal-safe? | Notes                                                           |
|----------------|:------------:|------------------------------------------------------------------|
| `name`         | Yes          | Standard HTML attribute. Maps to FormData key.                   |
| `min`          | Yes          | Standard HTML attribute.                                         |
| `max`          | Yes          | Standard HTML attribute.                                         |
| `value`        | Yes          | Sets the default/reset value. Also the initial rendered value.   |
| `step`         | Yes          | Standard HTML attribute.                                         |
| `label`        | Yes          | String attribute. No Twig escaping concerns.                     |
| `help-text`    | Yes          | Hyphenated attribute. Use `attributes.setAttribute('help-text', val)` in Drupal. |
| `hx-size`      | Yes          | Custom attribute with `hx-` prefix. Valid HTML. Use `attributes.setAttribute('hx-size', 'sm')`. |
| `disabled`     | Yes          | Boolean. Use `{{ disabled ? 'disabled' : '' }}` in Twig.         |
| `show-value`   | Yes          | Boolean. Use `{{ show_value ? 'show-value' : '' }}` in Twig.     |
| `show-ticks`   | Yes          | Boolean. Use `{{ show_ticks ? 'show-ticks' : '' }}` in Twig.     |
| `aria-valuetext` | Yes        | String. Standard ARIA attribute. Pass human-readable value labels.|

## Accessibility Notes

- Always provide a `label` attribute or `slot="label"` content. An unlabeled slider violates WCAG 4.1.2.
- For pain scales or other contextual ranges, use `aria-valuetext` to provide human-readable value descriptions (e.g., `"7 — Moderate-Severe"`).
- The slider is fully keyboard-accessible: arrow keys, Home, End, Page Up, Page Down.
- The `disabled` attribute prevents interaction and announces the disabled state to assistive technology.
