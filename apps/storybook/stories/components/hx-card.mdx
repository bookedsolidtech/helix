## Overview

The `hx-card` component is a foundational layout primitive in the HELIX design system. It provides a structured,
visually distinct container for presenting related information in clinical dashboards, patient summaries, and
administrative interfaces.

Cards support three visual variants (`default`, `featured`, `compact`) and three elevation levels (`flat`, `raised`, `floating`).
When given an `hx-href` attribute, cards become interactive and emit a custom `hx-card-click` event on activation.

### When to Use

- **Patient summaries** — Display patient demographics, vitals, and care team information
- **Dashboard widgets** — Show department metrics, pending tasks, or system alerts
- **Content grouping** — Organize related form fields, diagnostic results, or medication lists
- **Clickable navigation** — Create interactive tiles that navigate to detail pages

### When Not to Use

- **Simple text blocks** — Use semantic HTML (`<article>`, `<section>`) for basic content
- **Data tables** — Use proper table elements for tabular data
- **Form containers** — Use `<fieldset>` or `hx-form` for form grouping

## Accessibility

The Card component follows WCAG 2.1 AA guidelines and implements proper ARIA patterns:

- **Keyboard navigation** — When `hx-href` is set, the card is keyboard-focusable (`tabindex="0"`) and activates on Enter/Space
- **Screen reader support** — Interactive cards have `role="link"` and announce their purpose
- **Focus visible** — Focus ring is clearly visible for keyboard users
- **Semantic structure** — Uses proper heading hierarchy and ARIA labels where appropriate

## Properties

| Property    | Type                                   | Default     | Description                                                          |
| ----------- | -------------------------------------- | ----------- | -------------------------------------------------------------------- |
| `variant`   | `'default' \| 'featured' \| 'compact'` | `'default'` | Visual style variant of the card                                     |
| `elevation` | `'flat' \| 'raised' \| 'floating'`     | `'flat'`    | Shadow depth of the card                                             |
| `hx-href`   | `string`                               | `''`        | Optional URL for interactive cards. When set, card becomes clickable |

## Slots

| Slot        | Description                                                                     |
| ----------- | ------------------------------------------------------------------------------- |
| `image`     | Optional image at the top of the card (e.g., patient photo, facility thumbnail) |
| `heading`   | Card title or heading text                                                      |
| `(default)` | Main body content of the card                                                   |
| `footer`    | Footer content (timestamps, metadata, secondary info)                           |
| `actions`   | Action buttons rendered at the bottom of the card                               |

## CSS Custom Properties

Override these tokens to theme the card:

| Property                  | Default                       | Description          |
| ------------------------- | ----------------------------- | -------------------- |
| `--hx-card-bg`            | `var(--hx-color-neutral-0)`   | Background color     |
| `--hx-card-color`         | `var(--hx-color-neutral-800)` | Text color           |
| `--hx-card-border-color`  | `var(--hx-color-neutral-200)` | Border color         |
| `--hx-card-border-radius` | `var(--hx-border-radius-lg)`  | Border radius        |
| `--hx-card-padding`       | `var(--hx-space-6)`           | Internal padding     |
| `--hx-card-gap`           | `var(--hx-space-4)`           | Gap between sections |

## CSS Parts

Style internal card elements using `::part()`:

| Part      | Description                |
| --------- | -------------------------- |
| `card`    | The outer card container   |
| `image`   | The image slot container   |
| `heading` | The heading slot container |
| `body`    | The body slot container    |
| `footer`  | The footer slot container  |
| `actions` | The actions slot container |

## Events

| Event           | Detail            | Description                                                                          |
| --------------- | ----------------- | ------------------------------------------------------------------------------------ |
| `hx-card-click` | `{ url: string }` | Fired when an interactive card (with `hx-href`) is clicked or activated via keyboard |

## Usage Examples

### Basic Card

```html
<hx-card>
  <span slot="heading">Patient Summary</span>
  <p>Review vitals, lab results, and care plan updates.</p>
</hx-card>
```

### Interactive Card

```html
<hx-card hx-href="https://ehr.example.com/patient/12345" elevation="raised">
  <span slot="heading">James Wilson</span>
  <p>MRN: 00-12345 | DOB: 1958-03-22</p>
</hx-card>
```

### Full-Featured Card

```html
<hx-card variant="featured" elevation="raised">
  <img slot="image" src="/facility-photo.jpg" alt="Main campus entrance" />
  <span slot="heading">Facility Update</span>
  <p>The new ambulatory care wing is now open.</p>
  <span slot="footer">
    <small>Posted Feb 10, 2026</small>
  </span>
  <span slot="actions">
    <hx-button size="sm">View Floor Plan</hx-button>
  </span>
</hx-card>
```

## Design Guidelines

### Variant Selection

- **Default** — General-purpose cards for standard dashboard content
- **Featured** — High-priority items, critical alerts, or urgent notifications
- **Compact** — Dense layouts like sidebars or secondary panels

### Elevation Selection

- **Flat** — Inline content that blends with the page surface
- **Raised** — Standard elevation for primary dashboard cards
- **Floating** — High-priority content that appears above the surface (use sparingly)

### Content Guidelines

- Keep heading text concise (1-3 words)
- Use the footer slot for timestamps and metadata
- Limit action buttons to 1-3 per card
- Ensure sufficient color contrast (WCAG AA minimum)

## Drupal Integration

The `hx-card` component integrates seamlessly with Drupal 10/11 through Twig templates and Drupal behaviors. This section demonstrates property-driven and slot-driven patterns, CDN loading, and event handling.

### Twig Template Examples

**Property-driven approach (recommended for simple cards):**

```twig
{# templates/components/card.html.twig #}
<hx-card
  variant="{{ variant|default('default') }}"
  elevation="{{ elevation|default('flat') }}"
  {% if url %}hx-href="{{ url }}"{% endif %}
>
  {% if heading %}
    <span slot="heading">{{ heading }}</span>
  {% endif %}
  {{ content }}
  {% if footer %}
    <span slot="footer">{{ footer }}</span>
  {% endif %}
</hx-card>
```

**Slot-driven approach (for complex content with markup):**

```twig
{# templates/node/node--article--teaser.html.twig #}
<hx-card variant="featured" elevation="raised">
  {% if content.field_image|render %}
    <img
      slot="image"
      src="{{ file_url(node.field_image.entity.uri.value) }}"
      alt="{{ node.field_image.alt }}"
    />
  {% endif %}

  <span slot="heading">{{ node.label }}</span>

  <div class="card-body">
    {{ content.body }}
  </div>

  <div slot="footer">
    <small>{{ node.created.value|date('F j, Y') }}</small>
  </div>

  {% if node.url %}
    <div slot="actions">
      <hx-button size="sm" variant="secondary">Read More</hx-button>
    </div>
  {% endif %}
</hx-card>
```

### Drupal Behaviors Integration

Use Drupal.behaviors to initialize card event listeners and handle navigation:

```javascript
/**
 * @file
 * hx-card Drupal behavior.
 */

(function (Drupal, once) {
  'use strict';

  /**
   * Attach hx-card event listeners.
   */
  Drupal.behaviors.hxCard = {
    attach: function (context, settings) {
      // Initialize all interactive cards (with hx-href attribute)
      once('hx-card-init', 'hx-card[hx-href]', context).forEach((card) => {
        // Listen for card click events
        card.addEventListener('hx-card-click', (e) => {
          const url = e.detail.url;

          // Option 1: Standard navigation
          window.location.href = url;

          // Option 2: AJAX navigation (for single-page apps)
          // e.preventDefault();
          // Drupal.ajax({ url: url }).execute();

          // Option 3: Open in new tab for external links
          // if (url.startsWith('http')) {
          //   window.open(url, '_blank');
          // }
        });

        // Optional: Track card interactions for analytics
        card.addEventListener('hx-card-click', (e) => {
          if (typeof gtag !== 'undefined') {
            gtag('event', 'card_click', {
              card_url: e.detail.url,
              card_variant: card.getAttribute('variant'),
            });
          }
        });
      });

      // Optional: Add hover effects or data attributes
      once('hx-card-enhance', 'hx-card', context).forEach((card) => {
        // Add custom data attributes for theming
        const variant = card.getAttribute('variant');
        if (variant === 'featured') {
          card.dataset.priority = 'high';
        }
      });
    },

    detach: function (context, settings, trigger) {
      if (trigger === 'unload') {
        // Clean up event listeners on unload
        context.querySelectorAll('hx-card').forEach((card) => {
          card.removeEventListener('hx-card-click', () => {});
        });
      }
    },
  };
})(Drupal, once);
```

### CDN Loading Strategy

Add to `your_theme.libraries.yml`:

```yaml
# CDN approach (recommended for production)
helix-components:
  js:
    https://cdn.jsdelivr.net/npm/@helixui/library@1.0.0/dist/index.js:
      type: external
      attributes:
        type: module
  version: 1.0.0
  header: true
```

For local development with npm-installed components:

```yaml
# Local approach (for development or private hosting)
helix-components-local:
  js:
    /libraries/helix/dist/index.js:
      type: file
      attributes:
        type: module
  version: VERSION
  header: true
```

Attach the library to your theme or specific pages:

```yaml
# your_theme.info.yml
libraries:
  - your_theme/helix-components
```

Or conditionally in a template:

```twig
{{ attach_library('your_theme/helix-components') }}
```

### Performance Optimization

**Lazy loading for below-the-fold cards:**

```javascript
(function (Drupal, once) {
  Drupal.behaviors.hxCardLazy = {
    attach: function (context) {
      if ('IntersectionObserver' in window) {
        const observer = new IntersectionObserver((entries) => {
          entries.forEach((entry) => {
            if (entry.isIntersecting) {
              const card = entry.target;
              card.setAttribute('elevation', 'raised');
              observer.unobserve(card);
            }
          });
        });

        once('hx-card-lazy', 'hx-card[data-lazy]', context).forEach((card) => {
          observer.observe(card);
        });
      }
    },
  };
})(Drupal, once);
```
