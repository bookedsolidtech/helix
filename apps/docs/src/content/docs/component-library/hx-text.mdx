## Overview

`hx-text` is a semantic typography wrapper that applies consistent text styles using Helix design tokens. It renders an inner element (`span` by default) with the correct font size, weight, line height, letter spacing, and color for the chosen variant.

Use `hx-text` to display body copy, labels, captions, code references, and overlines in healthcare UI â€” patient names, field labels, clinical notes, status messages, and metadata all benefit from a single, token-driven text component.

**Use `hx-text` when:** rendering any presentational text that should follow the Helix type scale. **Use native heading elements (`h1`â€“`h6`) instead when:** the text needs to establish document hierarchy â€” `hx-text` is not a heading component.

## Basic Usage

Minimal HTML snippet â€” no build tool required:

```html
<!-- Default body text -->
<hx-text>Patient information updated successfully.</hx-text>

<!-- Label variant with semibold weight -->
<hx-text variant="label" weight="semibold">Date of Birth</hx-text>

<!-- Caption with subtle color -->
<hx-text variant="caption" color="subtle">Last reviewed on 2026-03-05</hx-text>

<!-- Single-line truncation (include attribute to enable) -->
<hx-text truncate>Long patient note that gets clipped at container width</hx-text>

<!-- Multi-line clamp (pass as string; component coerces to number) -->
<hx-text lines="3">Extended clinical notes spanning multiple lines...</hx-text>

<!-- Semantic element override -->
<hx-text as="p" variant="body">Paragraph-level body content.</hx-text>
<hx-text as="strong" variant="label">Bold label text</hx-text>
```

Set properties via JavaScript for dynamic control:

```javascript
const el = document.querySelector('hx-text');

// Change variant at runtime
el.variant = 'body-lg';

// Update color based on status
el.color = 'danger';

// Enable truncation
el.truncate = true;

// Clamp to 2 lines
el.lines = 2;
```

## Properties

| Property   | Attribute  | Type                                                                                             | Default     | Description                                                                                                                       |
| ---------- | ---------- | ------------------------------------------------------------------------------------------------ | ----------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `variant`  | `variant`  | `'body' \| 'body-sm' \| 'body-lg' \| 'label' \| 'label-sm' \| 'caption' \| 'code' \| 'overline'` | `'body'`    | Typography variant controlling font size, line height, and letter spacing. Reflects to attribute.                                 |
| `weight`   | `weight`   | `'regular' \| 'medium' \| 'semibold' \| 'bold' \| undefined`                                     | `undefined` | Font weight override. When unset, the variant's default weight is used. Reflects to attribute.                                    |
| `color`    | `color`    | `'default' \| 'subtle' \| 'disabled' \| 'inverse' \| 'danger' \| 'success' \| 'warning'`         | `'default'` | Semantic color intent. Reflects to attribute.                                                                                     |
| `truncate` | `truncate` | `boolean`                                                                                        | `false`     | Clips text to a single line with an ellipsis overflow. Include the attribute to enable. Reflects to attribute.                    |
| `lines`    | `lines`    | `number`                                                                                         | `0`         | Maximum number of lines before clamping with ellipsis. Overrides `truncate` when set. 0 disables clamping. Reflects to attribute. |
| `as`       | `as`       | `'span' \| 'p' \| 'strong' \| 'em' \| 'div'`                                                     | `'span'`    | The HTML element to render as the inner base element. Use to produce semantically appropriate markup. Reflects to attribute.      |

### Variant Reference

| Variant    | Font Size | Weight   | Line Height | Use Case                                      |
| ---------- | --------- | -------- | ----------- | --------------------------------------------- |
| `body`     | 1rem      | Regular  | 1.5         | Standard body copy, descriptions              |
| `body-sm`  | 0.875rem  | Regular  | 1.5         | Secondary content, metadata                   |
| `body-lg`  | 1.125rem  | Regular  | 1.5         | Introductory text, highlighted summaries      |
| `label`    | 1rem      | Medium   | 1.25        | Form field labels, section headers            |
| `label-sm` | 0.875rem  | Medium   | 1.25        | Compact labels, table column headers          |
| `caption`  | 0.75rem   | Regular  | 1.5         | Supporting copy, timestamps, helper text      |
| `code`     | 0.875rem  | Regular  | 1.5         | Monospace content: ICD codes, MRNs, snippets  |
| `overline` | 0.75rem   | Semibold | 1.25        | Section dividers, category labels (uppercase) |

## CSS Custom Properties

| Property                   | Default                                      | Description                               |
| -------------------------- | -------------------------------------------- | ----------------------------------------- |
| `--hx-text-font-size`      | Set per variant (e.g. `1rem` for `body`)     | Font size. Override to set a custom size. |
| `--hx-text-font-weight`    | Set per variant (e.g. `400` for `body`)      | Font weight. Override per instance.       |
| `--hx-text-line-height`    | Set per variant (e.g. `1.5` for `body`)      | Line height. Override per instance.       |
| `--hx-text-letter-spacing` | Set per variant (e.g. `0` for `body`)        | Letter spacing. Override per instance.    |
| `--hx-text-color`          | Set per color (e.g. `#0f172a` for `default`) | Text color. Override per instance.        |

Override at the semantic token level to theme all instances, or at the component level for a single instance:

```css
/* Override font size via semantic token (affects all body-variant text) */
:root {
  --hx-font-size-md: 1.0625rem;
}

/* Override a single instance inline */
hx-text#patient-name {
  --hx-text-font-size: 1.25rem;
  --hx-text-font-weight: 600;
}
```

## Accessibility

`hx-text` is a presentational component. It exposes no ARIA roles of its own â€” semantic meaning comes from the `as` property and the surrounding document structure.

| Topic         | Details                                                                                                                                                                                                                      |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ARIA role     | No ARIA role is applied. The rendered element inherits the implicit role of its tag (e.g. `span` â†’ generic, `strong` â†’ strong importance, `em` â†’ emphasis).                                                                  |
| Keyboard      | `hx-text` is not interactive and does not receive keyboard focus.                                                                                                                                                            |
| Screen reader | Rendered text is read as normal document content. When truncation is active (`truncate` or `lines > 0`), a `title` attribute is automatically added to the inner element exposing the full unclipped text to screen readers. |
| Truncation    | Full text is exposed via `title` on the base element when `truncate` or `lines > 0` is active. Assistive technology users can access the complete string even when the visible text is clipped.                              |
| Color         | `color="disabled"` renders intentionally low-contrast text â€” WCAG 1.4.3 exempts inactive UI components. `color="inverse"` must be placed on a sufficiently dark background to meet contrast requirements.                    |
| WCAG          | Meets WCAG 2.1 AA. Zero axe-core violations across all variant and color combinations. SC 1.3.1, SC 1.4.3, SC 4.1.2 satisfied.                                                                                               |

### Semantic Element Best Practices

Use the `as` property to produce semantically correct markup:

```html
<!-- Inline content (default) -->
<hx-text variant="body">Inline patient data</hx-text>

<!-- Block-level paragraph -->
<hx-text as="p" variant="body">Clinical notes paragraph content.</hx-text>

<!-- Emphasized text -->
<hx-text as="em" variant="body" color="subtle">Optional field</hx-text>

<!-- Strong importance -->
<hx-text as="strong" variant="label" weight="semibold">CRITICAL ALERT</hx-text>
```

## Drupal Integration

Use the component in a Twig template after registering the library:

```twig
{# my-module/templates/patient-record.html.twig #}

{# Standard body text #}
<hx-text variant="body" color="default">{{ patient.name }}</hx-text>

{# Label with weight override #}
<hx-text variant="label" weight="semibold">Date of Birth</hx-text>

{# Caption with subtle color #}
<hx-text variant="caption" color="subtle">{{ node.field_last_reviewed }}</hx-text>

{# Boolean attribute (truncate) â€” include attribute to enable, omit to disable #}
{% if truncate_notes %}
  <hx-text truncate>{{ clinical_notes }}</hx-text>
{% else %}
  <hx-text>{{ clinical_notes }}</hx-text>
{% endif %}

{# Numeric attribute (lines) â€” pass as string; component coerces to number #}
<hx-text lines="3">{{ clinical_summary }}</hx-text>

{# Semantic element override #}
<hx-text as="p" variant="body">{{ body_text }}</hx-text>
<hx-text as="strong" variant="label">{{ field_label }}</hx-text>

{# Status coloring #}
<hx-text variant="body" color="{{ status == 'critical' ? 'danger' : 'success' }}">
  {{ status_label }}
</hx-text>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

`hx-text` is a display-only component with no events. For dynamic color or content updates, use standard JavaScript:

```javascript
Drupal.behaviors.helixText = {
  attach(context) {
    // once() is a Drupal core utility (no import needed) â€” prevents duplicate event binding during AJAX attach cycles
    once('helixText', '[data-hx-text-status]', context).forEach((el) => {
      const textEl = el.querySelector('hx-text');
      if (textEl) {
        textEl.color = el.dataset.hxTextStatus === 'critical' ? 'danger' : 'default';
      }
    });
  },
};
```
