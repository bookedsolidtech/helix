## Overview

`hx-split-panel` is a resizable two-pane layout component with a draggable divider. It splits available space between a start panel and an end panel, with full keyboard navigation, pointer drag support, snap points, min/max constraints, and optional panel collapse affordances.

The component is designed for enterprise healthcare UIs where flexible workspace layouts are essential — patient list / detail views, navigation sidebars, document preview panes, and clinical dashboard arrangements. The draggable divider carries `role="separator"` with full ARIA slider semantics, ensuring assistive technology users can resize panels via keyboard.

**Use `hx-split-panel` when:** you need a resizable two-pane layout where the user can adjust how space is divided between two content areas.

**Use separate containers with fixed widths instead when:** the layout ratio is predetermined and should never be user-adjustable.

## Basic Usage

```html
<!-- Horizontal split (default) -->
<hx-split-panel position="50">
  <div slot="start">Navigation or sidebar content</div>
  <div slot="end">Main content area</div>
</hx-split-panel>

<!-- Vertical split -->
<hx-split-panel orientation="vertical" position="40">
  <div slot="start">Top panel content</div>
  <div slot="end">Bottom panel content</div>
</hx-split-panel>

<!-- With min/max constraints and snap points -->
<hx-split-panel position="50" min="20" max="80" snap="[25,50,75]">
  <div slot="start">Left panel</div>
  <div slot="end">Right panel</div>
</hx-split-panel>

<!-- Collapsible panels -->
<hx-split-panel position="35" collapsible>
  <div slot="start">Sidebar</div>
  <div slot="end">Content</div>
</hx-split-panel>
```

## Properties

| Property           | Attribute            | Type                         | Default        | Description                                                                                                                    |
| ------------------ | -------------------- | ---------------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `position`         | `position`           | `number`                     | `50`           | Position of the divider as a percentage (0–100) of the start panel. Reflects to attribute.                                     |
| `positionInPixels` | `position-in-pixels` | `number \| undefined`        | `undefined`    | Position of the divider in pixels. Converted to percentage after first paint. Overrides `position` on initial render.          |
| `orientation`      | `orientation`        | `'horizontal' \| 'vertical'` | `'horizontal'` | Orientation of the split. Reflects to attribute.                                                                               |
| `min`              | `min`                | `number`                     | `0`            | Minimum position as a percentage. Prevents full collapse of start panel. Reflects to attribute.                                |
| `max`              | `max`                | `number`                     | `100`          | Maximum position as a percentage. Prevents full expansion of start panel. Reflects to attribute.                               |
| `snap`             | `snap`               | `number[]`                   | `[]`           | Snap points as an array of percentages. Divider snaps within a 5% threshold. Accepts JSON string in HTML: `snap="[25,50,75]"`. |
| `disabled`         | `disabled`           | `boolean`                    | `false`        | When true, the divider cannot be dragged. Reflects to attribute.                                                               |
| `collapsible`      | `collapsible`        | `boolean`                    | `false`        | When true, collapse/expand buttons appear on the divider. Reflects to attribute.                                               |
| `collapsed`        | `collapsed`          | `'start' \| 'end' \| null`   | `null`         | Which panel is collapsed. Set to `'start'` or `'end'` to collapse programmatically. Reflects to attribute.                     |

## Events

| Event           | Detail Type            | Description                                                                                                      |
| --------------- | ---------------------- | ---------------------------------------------------------------------------------------------------------------- |
| `hx-reposition` | `{ position: number }` | Fired when the divider is moved (drag or keyboard). Bubbles and composed. Not fired if position does not change. |

```javascript
const panel = document.querySelector('hx-split-panel');

panel.addEventListener('hx-reposition', (e) => {
  console.log('Divider moved to:', e.detail.position, '%');
  // Persist the position to user preferences
  localStorage.setItem('panelPosition', e.detail.position);
});
```

## Accessibility

`hx-split-panel` implements the ARIA splitter pattern. The divider carries `role="separator"` with full range semantics (`aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-orientation`) making it operable by assistive technology users via keyboard.

| Topic              | Details                                                                                                                                                                                                                      |
| ------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------- |
| ARIA role          | The divider has `role="separator"`. This is the correct role for a splitter bar per the ARIA APG. `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, and `aria-orientation` are set automatically from component properties. |
| Accessible name    | The divider has `aria-label="Resize panels"` ensuring screen reader users understand its purpose.                                                                                                                            |
| `aria-orientation` | Set to `"vertical"` for a horizontal split (the bar is vertical), and `"horizontal"` for a vertical split (the bar is horizontal). This matches the ARIA specification for separator orientation.                            |
| `aria-disabled`    | Set to `"true"` when `disabled` is true. Omitted entirely when enabled (not set to `"false"`).                                                                                                                               |
| Keyboard focus     | The divider is focusable (`tabindex="0"`) when enabled. When disabled, `tabindex="-1"` prevents keyboard access.                                                                                                             |
| Focus indicator    | `:focus-visible` applies a `2px` solid outline with a `4px` color-mix box shadow, meeting WCAG 2.4.7 (Focus Visible) and WCAG 2.4.11 (Focus Appearance) requirements.                                                        |
| Collapse buttons   | When `collapsible` is enabled, collapse/expand buttons have descriptive `aria-label` values: `"Collapse start panel"`, `"Collapse end panel"`, and `"Expand [start                                                           | end] panel"`. |
| WCAG               | Meets WCAG 2.1 AA. SC 2.1.1 (Keyboard), SC 2.4.7 (Focus Visible), SC 4.1.2 (Name, Role, Value), SC 1.4.11 (Non-text Contrast). Zero axe-core violations across all variants.                                                 |

## Keyboard Navigation

The divider is keyboard operable once focused. All resize operations support the standard ARIA range widget key bindings:

| Key          | Behavior                                           |
| ------------ | -------------------------------------------------- |
| `Tab`        | Moves focus to / from the divider.                 |
| `ArrowRight` | Increases position by 1% (horizontal orientation). |
| `ArrowDown`  | Increases position by 1% (vertical orientation).   |
| `ArrowLeft`  | Decreases position by 1% (horizontal orientation). |
| `ArrowUp`    | Decreases position by 1% (vertical orientation).   |
| `PageUp`     | Increases position by 10%.                         |
| `PageDown`   | Decreases position by 10%.                         |
| `Home`       | Moves divider to the `min` value.                  |
| `End`        | Moves divider to the `max` value.                  |

All keyboard moves are clamped to `[min, max]` and apply snap logic when snap points are configured. No keyboard interaction occurs when `disabled` is true.

## Drupal Integration

```twig
{# my-module/templates/patient-layout.html.twig #}
<hx-split-panel
  position="{{ position|default(50) }}"
  orientation="{{ orientation|default('horizontal') }}"
  min="{{ min|default(0) }}"
  max="{{ max|default(100) }}"
  {% if snap %}snap="{{ snap }}"{% endif %}
  {% if collapsible %}collapsible{% endif %}
  {% if collapsed %}collapsed="{{ collapsed }}"{% endif %}
  {% if disabled %}disabled{% endif %}
>
  <div slot="start">{{ start_content }}</div>
  <div slot="end">{{ end_content }}</div>
</hx-split-panel>
```

**Note on the `snap` property:** Pass snap points as a JSON array string: `snap="[25,50,75]"`. The component's custom attribute converter handles both JSON arrays and comma-separated strings.

**JS-only properties:** `positionInPixels` and programmatically-set `snap` arrays require JavaScript and cannot be set purely via Twig attributes. Use `position` (percentage) and `snap` (JSON string) for Twig-only rendering.

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

Listen for resize events in Drupal behaviors:

```javascript
Drupal.behaviors.helixSplitPanel = {
  attach(context) {
    once('helixSplitPanel', 'hx-split-panel', context).forEach((el) => {
      el.addEventListener('hx-reposition', (e) => {
        const { position } = e.detail;
        // Persist panel position to user preferences via AJAX
        Drupal.ajax({
          url: '/api/preferences/panel-position',
          base: false,
          element: el,
        }).execute({ position });
      });
    });
  },
};
```
