## Overview

`hx-split-button` combines a primary action button with a dropdown menu for secondary actions. It implements the [ARIA menu button pattern](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/) with full keyboard navigation and screen reader support.

The primary button fires `hx-click` for the main action. The chevron trigger opens a dropdown populated with `hx-menu-item` children. Selecting a menu item fires `hx-select` with `value` and `label` in the event detail.

**Use `hx-split-button` when:**

- There is one clear primary action and several secondary options (e.g., Save / Save as Draft / Save & Publish).
- You need to reduce toolbar clutter while keeping secondary actions discoverable.
- Healthcare context: Save Record with options to Save Draft, Notify Team, or Archive.

**Use `hx-button` + a separate `hx-menu` instead when:** the primary and secondary actions are equally weighted or you need more compositional control over the layout.

## Installation

```bash
# Full library
npm install @helix/library

# Or import only this component (tree-shaking friendly)
import '@helix/library/components/hx-split-button';
```

`hx-split-button` uses `hx-menu-item` for its dropdown options. Both are included in `@helix/library`.

## Basic Usage

```html
<!-- Label via property (recommended) -->
<hx-split-button label="Save Record">
  <hx-menu-item slot="menu" value="save-draft">Save as Draft</hx-menu-item>
  <hx-menu-item slot="menu" value="save-publish">Save &amp; Publish</hx-menu-item>
  <hx-menu-item slot="menu" value="save-notify">Save &amp; Notify Team</hx-menu-item>
</hx-split-button>

<!-- Label via default slot -->
<hx-split-button>
  Save Record
  <hx-menu-item slot="menu" value="save-draft">Save as Draft</hx-menu-item>
</hx-split-button>

<!-- Danger variant for destructive actions -->
<hx-split-button variant="danger" label="Delete Record">
  <hx-menu-item slot="menu" value="archive">Archive Record</hx-menu-item>
  <hx-menu-item slot="menu" value="flag">Flag for Review</hx-menu-item>
</hx-split-button>

<script>
  const btn = document.querySelector('hx-split-button');

  // Primary button click
  btn.addEventListener('hx-click', (e) => {
    console.log('Primary action triggered', e.detail.originalEvent);
  });

  // Menu item selection
  btn.addEventListener('hx-select', (e) => {
    console.log('Selected:', e.detail.value, e.detail.label);
  });
</script>
```

## Properties

| Property       | Attribute       | Type                                                                         | Default               | Description                                                                   |
| -------------- | --------------- | ---------------------------------------------------------------------------- | --------------------- | ----------------------------------------------------------------------------- |
| `variant`      | `variant`       | `'primary' \| 'secondary' \| 'tertiary' \| 'danger' \| 'ghost' \| 'outline'` | `'primary'`           | Visual style variant.                                                         |
| `size`         | `hx-size`       | `'sm' \| 'md' \| 'lg'`                                                       | `'md'`                | Size of both the primary button and the trigger. Rendered via `hx-size` attr. |
| `disabled`     | `disabled`      | `boolean`                                                                    | `false`               | Disables both the primary button and the dropdown trigger.                    |
| `label`        | `label`         | `string \| undefined`                                                        | `undefined`           | Primary button label. When set, overrides the default slot content.           |
| `triggerLabel` | `trigger-label` | `string`                                                                     | `'More actions'`      | Accessible label for the dropdown trigger button. Override for localization.  |
| `menuLabel`    | `menu-label`    | `string`                                                                     | `'Secondary actions'` | Accessible label for the dropdown menu panel. Override for localization.      |

## Events

| Event       | Detail Type                        | Description                                                                                                                                         |
| ----------- | ---------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `hx-click`  | `{ originalEvent: MouseEvent }`    | Dispatched when the primary action button is clicked and is not disabled. Bubbles and composed.                                                     |
| `hx-select` | `{ value: string; label: string }` | Dispatched when a menu item is selected. `value` is from the `hx-menu-item`'s `value` attribute; `label` is its visible text. Bubbles and composed. |

## CSS Parts

| Part      | Description                                                                      |
| --------- | -------------------------------------------------------------------------------- |
| `button`  | The primary action `<button>` element. Target for custom label styling.          |
| `trigger` | The dropdown trigger `<button>` element containing the chevron icon.             |
| `menu`    | The dropdown panel `<div>` with `role="menu"`. Target for positioning overrides. |

## Slots

| Slot        | Description                                                                                                 |
| ----------- | ----------------------------------------------------------------------------------------------------------- |
| _(default)_ | Primary button label text. Used when the `label` property is not set.                                       |
| `menu`      | `hx-menu-item` children rendered inside the dropdown panel. Each item must have a unique `value` attribute. |

## Accessibility

`hx-split-button` implements the [ARIA Menu Button pattern](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/) (WAI-ARIA Authoring Practices Guide). The dropdown trigger is a `<button>` with `aria-haspopup="menu"` and `aria-expanded` reflecting the open state.

| Topic              | Details                                                                                                                                                                           |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ARIA pattern       | Menu button pattern: trigger has `aria-haspopup="menu"` + `aria-expanded`; panel has `role="menu"` + `aria-label`.                                                                |
| `aria-controls`    | The trigger's `aria-controls` points to the menu panel `id` for explicit ownership.                                                                                               |
| `aria-label`       | Trigger defaults to `"More actions"`; menu panel defaults to `"Secondary actions"`. Both are overridable via `trigger-label` / `menu-label`.                                      |
| Keyboard — primary | `Tab` to focus; `Enter` / `Space` to activate; `ArrowDown` opens the menu and focuses first item.                                                                                 |
| Keyboard — trigger | `Tab` to focus; `Enter` / `Space` / click to open menu; `ArrowDown` opens menu.                                                                                                   |
| Keyboard — menu    | `ArrowDown` / `ArrowUp` cycle through enabled items (wraps); `Home` goes to first item; `End` goes to last item; `Escape` closes and returns focus to trigger; `Tab` closes menu. |
| Disabled items     | `hx-menu-item[disabled]` items are excluded from keyboard navigation and announce as unavailable.                                                                                 |
| Focus management   | Opening the menu focuses the first enabled item. Closing via `Escape` or item selection returns focus to the trigger.                                                             |
| Reduced motion     | All transitions and the open animation are disabled when `prefers-reduced-motion: reduce` is active.                                                                              |
| WCAG               | Meets WCAG 2.1 AA. SC 2.1.1 (Keyboard), SC 4.1.2 (Name, Role, Value), SC 1.4.11 (Non-text Contrast). Zero axe-core violations.                                                    |

## Drupal Integration

```twig
{# my-module/templates/patient-record-actions.html.twig #}
<hx-split-button
  variant="{{ variant|default('primary') }}"
  label="{{ label|default('Save Record') }}"
  trigger-label="{{ trigger_label|default('More actions') }}"
  menu-label="{{ menu_label|default('Secondary actions') }}"
  {% if disabled %}disabled{% endif %}
>
  {% for item in menu_items %}
    <hx-menu-item
      slot="menu"
      value="{{ item.value }}"
      {% if item.disabled %}disabled{% endif %}
    >{{ item.label }}</hx-menu-item>
  {% endfor %}
</hx-split-button>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

Handle split button events in Drupal behaviors using `once()` to prevent duplicate listeners on AJAX:

```javascript
Drupal.behaviors.helixSplitButton = {
  attach(context) {
    once('helixSplitButton', 'hx-split-button', context).forEach((el) => {
      el.addEventListener('hx-click', () => {
        // Primary action: save record
        Drupal.ajax({ url: el.dataset.saveUrl }).execute();
      });

      el.addEventListener('hx-select', (e) => {
        const { value, label } = e.detail;
        // Secondary action: route by value
        switch (value) {
          case 'save-draft':
            Drupal.ajax({ url: el.dataset.draftUrl }).execute();
            break;
          case 'save-notify':
            Drupal.ajax({ url: el.dataset.notifyUrl }).execute();
            break;
          default:
            console.warn('Unknown split button action:', value, label);
        }
      });
    });
  },
};
```

## Standalone HTML Example

Copy-paste into a `.html` file and open in any browser — no build tool required:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>hx-split-button example</title>
    <!-- @helix/library is a private package — install via npm workspace, not CDN -->
    <!-- In your project: import '@helix/library'; in your bundler entry point -->
    <style>
      body {
        font-family: sans-serif;
        padding: 2rem;
        display: flex;
        flex-direction: column;
        gap: 1.5rem;
      }
      #status {
        padding: 0.5rem 1rem;
        border-radius: 0.375rem;
        background: #f1f5f9;
        font-size: 0.875rem;
        color: #334155;
        min-height: 2.5rem;
        display: flex;
        align-items: center;
      }
    </style>
  </head>
  <body>
    <h2>Patient Record Actions</h2>

    <hx-split-button id="record-btn" label="Save Record">
      <hx-menu-item slot="menu" value="save-draft">Save as Draft</hx-menu-item>
      <hx-menu-item slot="menu" value="save-publish">Save &amp; Publish</hx-menu-item>
      <hx-menu-item slot="menu" value="save-notify">Save &amp; Notify Team</hx-menu-item>
    </hx-split-button>

    <hx-split-button variant="danger" label="Delete Record">
      <hx-menu-item slot="menu" value="archive">Archive Record</hx-menu-item>
      <hx-menu-item slot="menu" value="flag">Flag for Review</hx-menu-item>
    </hx-split-button>

    <div id="status">Interact with the split buttons above to see events.</div>

    <script>
      const status = document.getElementById('status');
      const recordBtn = document.getElementById('record-btn');

      recordBtn.addEventListener('hx-click', (e) => {
        status.textContent = 'Primary action: Save Record clicked.';
      });

      document.querySelectorAll('hx-split-button').forEach((btn) => {
        btn.addEventListener('hx-select', (e) => {
          status.textContent = `Menu item selected — value: "${e.detail.value}", label: "${e.detail.label}"`;
        });
      });
    </script>
  </body>
</html>
```
