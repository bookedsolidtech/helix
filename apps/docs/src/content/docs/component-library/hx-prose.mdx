## Overview

`hx-prose` is a Light DOM typography wrapper for rich text content produced by CMS editors, Markdown renderers, or any source that outputs structured HTML body copy. It injects scoped typographic styles — headings, paragraphs, lists, blockquotes, tables, code blocks, and media — without leaking those styles to the rest of the page.

Unlike most components in this library, `hx-prose` intentionally does **not** use Shadow DOM. This makes it the correct choice for CMS-injected content where axe-core accessibility scanning, global heading hierarchy, and Drupal CKEditor output all need to remain fully accessible to the DOM. For purely decorative or self-contained UI, use `hx-card` or `hx-container` instead.

Use `hx-prose` when wrapping:

- CKEditor or WYSIWYG output from Drupal
- Markdown-rendered HTML from static content pipelines
- Patient care documentation, clinical protocols, or any structured body copy

## Basic Usage

Minimal HTML snippet — no build tool required:

```html
<hx-prose>
  <h2>Patient Care Guidelines</h2>
  <p>Effective patient care requires a systematic approach to assessment and planning.</p>
  <ul>
    <li>Patient safety is the top priority</li>
    <li>Evidence-based practice guides all decisions</li>
  </ul>
</hx-prose>
```

With size and max-width:

```html
<hx-prose size="lg" max-width="72ch">
  <h1>Clinical Protocol Overview</h1>
  <p>This document outlines the standard operating procedures for the ICU.</p>
</hx-prose>
```

## Properties

| Property   | Attribute   | Type                     | Default  | Description                                                                                                                 |
| ---------- | ----------- | ------------------------ | -------- | --------------------------------------------------------------------------------------------------------------------------- |
| `size`     | `size`      | `'sm' \| 'base' \| 'lg'` | `'base'` | Typography scale for the prose content. Sets `--hx-prose-font-size` to the corresponding size token.                        |
| `maxWidth` | `max-width` | `string`                 | `''`     | Maximum content width. Accepts any valid CSS width value (`'640px'`, `'80ch'`, `'100%'`). Overrides `--hx-prose-max-width`. |

## Events

`hx-prose` is a transparent layout container. It dispatches no custom events.

## CSS Parts

`hx-prose` renders in the **Light DOM** (no Shadow DOM). CSS `::part()` selectors do not apply because there is no shadow root. Theming is accomplished via:

1. **CSS custom properties** on `hx-prose` or any ancestor (see table above)
2. **Standard CSS descendant selectors** — `hx-prose p`, `hx-prose h2`, etc.

This is intentional: Light DOM rendering ensures CMS-injected content (Drupal CKEditor output) remains fully styleable and accessible without shadow boundary constraints.

## Slots

| Slot        | Description                                                                                    |
| ----------- | ---------------------------------------------------------------------------------------------- |
| _(default)_ | Rich text content: headings, paragraphs, lists, tables, blockquotes, code blocks, images, etc. |

`hx-prose` is a transparent container — all slotted content renders directly in the Light DOM, making it selectable by standard CSS and scannable by accessibility tools including axe-core.

## Accessibility

| Topic         | Details                                                                                                                                                                                                                                      |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ARIA role     | None — `hx-prose` is a generic block container (`display: block`). It does not add any ARIA role. Content within it must supply their own appropriate semantics (`<h1>`–`<h6>`, `<table>`, `<ul>`, etc.).                                    |
| Keyboard      | `hx-prose` is not interactive and adds no keyboard behavior. Standard browser keyboard navigation applies to slotted links, form controls, and focusable elements.                                                                           |
| Screen reader | Slotted content is read in DOM order. Table captions are positioned `top` (default) so screen reader announcement order matches visual order — compliant with WCAG 2.1 Technique H39.                                                        |
| Focus         | No focus management. Focusable descendants (links, buttons, inputs) retain their natural tab order.                                                                                                                                          |
| Light DOM     | `hx-prose` uses Light DOM deliberately so axe-core can scan all child content without shadow boundary limitations. All accessibility violations inside `hx-prose` — such as `<img>` without `alt` — are detectable and reported by axe-core. |
| Images        | Consumer responsibility: every `<img>` inside `hx-prose` must have an `alt` attribute. Decorative images use `alt=""`. Informational images require descriptive alt text. Missing `alt` is a WCAG 2.1 AA violation (SC 1.1.1).               |
| Line height   | Body copy enforces a minimum 1.5 line-height ratio via `--hx-prose-line-height` — a healthcare readability mandate for patient-facing content.                                                                                               |
| WCAG          | Meets WCAG 2.1 AA for typography container requirements. Zero axe-core violations for well-formed content.                                                                                                                                   |

## Drupal Integration

`hx-prose` is purpose-built for Drupal CKEditor output. Wrap any text field or body field output in `hx-prose` to apply consistent typographic styles.

```twig
{# my-module/templates/node--article.html.twig #}

<hx-prose>
  {{ content.body }}
</hx-prose>
```

With size variant for a callout block:

```twig
<hx-prose size="sm">
  {{ content.field_sidebar_text }}
</hx-prose>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

`hx-prose` dispatches no events and requires no Drupal behaviors for standard usage. If dynamic content updates are required (e.g., AJAX-loaded body copy), replace the inner content directly — the component will re-apply all prose styles automatically:

```javascript
Drupal.behaviors.proseUpdate = {
  attach(context) {
    // hx-prose styles are applied via adopted stylesheets — no re-init needed.
    // Simply update the DOM content and styles remain active.
    once('proseUpdate', 'hx-prose[data-live]', context).forEach((el) => {
      el.innerHTML = newContent;
    });
  },
};
```
