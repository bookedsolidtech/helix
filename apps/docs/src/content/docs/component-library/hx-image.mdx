## Overview

`hx-image` is an accessible image wrapper that enforces alt-text discipline, handles image load failures gracefully, and integrates natively with Drupal responsive image styles. It wraps the browser's `<img>` element inside a `<figure>`/`<figcaption>` structure, providing slot-based caption support, aspect-ratio locking, and a CSS Parts API for external styling.

**Use `hx-image` when:** you need an image that is accessible by default, handles broken sources gracefully, supports responsive `srcset`/`sizes`, and integrates cleanly with Drupal Twig templates.

**Key behaviors:**

- Alt text is required for informative images. Use the `decorative` prop to explicitly mark decorative images — this sets `alt=""` and `role="presentation"` on the inner `<img>`.
- When an image fails to load, the component switches to a `fallback-src` if provided, then falls back to the `fallback` slot, announcing the failure via `role="alert"` to screen readers.
- Aspect ratio is locked via CSS `aspect-ratio` using the `ratio` attribute, preventing layout shift during image load.
- Lazy loading is enabled by default (`loading="lazy"`).

## Basic Usage

```html
<!-- Informative image with alt text -->
<hx-image
  src="/images/patient-photo.jpg"
  alt="Dr. Sarah Chen, Cardiology"
  ratio="1"
  rounded="50%"
></hx-image>

<!-- Decorative image — hidden from screen readers -->
<hx-image src="/images/hero-background.jpg" decorative ratio="21/9"></hx-image>

<!-- Image with caption -->
<hx-image src="/images/anatomy-diagram.png" alt="Diagram of the human heart" ratio="4/3">
  <span slot="caption">Figure 1: Anterior view of the human heart. Source: NIH.</span>
</hx-image>

<!-- Fallback on error -->
<hx-image
  src="/images/scan.jpg"
  fallback-src="/images/scan-placeholder.jpg"
  alt="MRI scan result"
  ratio="1"
>
  <span slot="fallback">Scan image unavailable</span>
</hx-image>

<!-- Responsive image with srcset -->
<hx-image
  src="/images/hero-800.jpg"
  srcset="/images/hero-400.jpg 400w, /images/hero-800.jpg 800w, /images/hero-1200.jpg 1200w"
  sizes="(max-width: 640px) 100vw, (max-width: 1024px) 80vw, 1200px"
  alt="Hospital campus exterior"
  ratio="16/9"
></hx-image>
```

## Properties

| Property      | Attribute      | Type                                                                    | Default     | Description                                                                                                                                 |
| ------------- | -------------- | ----------------------------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| `src`         | `src`          | `string`                                                                | `''`        | URL of the image to display.                                                                                                                |
| `alt`         | `alt`          | `string \| undefined`                                                   | `undefined` | Accessible text description. Required for informative images. Use `decorative` instead of `alt=""` to make decorative intent explicit.      |
| `decorative`  | `decorative`   | `boolean`                                                               | `false`     | Marks the image as decorative. Sets `alt=""` and `role="presentation"` on the inner `<img>`. Takes precedence over `alt` when both are set. |
| `loading`     | `loading`      | `'lazy' \| 'eager'`                                                     | `'lazy'`    | Loading strategy. `lazy` defers loading until the image enters the viewport. `eager` loads immediately.                                     |
| `fit`         | `fit`          | `'contain' \| 'cover' \| 'fill' \| 'none' \| 'scale-down' \| undefined` | `undefined` | CSS `object-fit` behavior. Controls how the image fills its container.                                                                      |
| `ratio`       | `ratio`        | `string \| undefined`                                                   | `undefined` | CSS `aspect-ratio` value (e.g. `"16/9"`, `"1"`, `"4/3"`). Prevents layout shift during image load.                                          |
| `rounded`     | `rounded`      | `boolean \| string \| undefined`                                        | `undefined` | Border radius. Boolean/empty string applies the theme's medium radius token. A CSS string (e.g. `"1rem"`, `"50%"`) is used directly.        |
| `width`       | `width`        | `number \| string \| undefined`                                         | `undefined` | Width of the image element. Numbers are converted to `px`. Used for layout hints and CLS prevention.                                        |
| `height`      | `height`       | `number \| string \| undefined`                                         | `undefined` | Height of the image element. Numbers are converted to `px`.                                                                                 |
| `fallbackSrc` | `fallback-src` | `string \| undefined`                                                   | `undefined` | Fallback image URL shown when the primary `src` fails to load. If this also fails, the `fallback` slot is shown.                            |
| `srcset`      | `srcset`       | `string \| undefined`                                                   | `undefined` | Comma-separated list of image candidates for responsive images. Passed directly to the inner `<img>` `srcset` attribute.                    |
| `sizes`       | `sizes`        | `string \| undefined`                                                   | `undefined` | Media conditions for responsive images. Works in conjunction with `srcset`.                                                                 |

## Events

| Event      | Detail | Description                                                                          |
| ---------- | ------ | ------------------------------------------------------------------------------------ |
| `hx-load`  | —      | Dispatched when the image has successfully loaded.                                   |
| `hx-error` | —      | Dispatched when the image fails to load — including after `fallback-src` also fails. |

```javascript
const img = document.querySelector('hx-image');

img.addEventListener('hx-load', () => {
  console.log('Image loaded successfully');
});

img.addEventListener('hx-error', () => {
  console.warn('Image failed to load — fallback displayed');
});
```

## Slots

| Slot       | Description                                                                                                                                                                              |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `fallback` | Content displayed when the image fails to load and no `fallback-src` is set (or when `fallback-src` also fails). The error container has `role="alert"` for screen reader announcements. |
| `caption`  | Optional caption content rendered in a `<figcaption>` element below the image. Only visible when content is provided.                                                                    |

```html
<!-- Fallback slot -->
<hx-image src="/images/scan.jpg" alt="MRI scan">
  <span slot="fallback">Scan unavailable. Contact imaging.</span>
</hx-image>

<!-- Caption slot -->
<hx-image src="/images/diagram.png" alt="Cardiac anatomy diagram">
  <span slot="caption">Figure 3A: Cardiac anatomy. Reproduced with permission.</span>
</hx-image>

<!-- Both slots -->
<hx-image src="/images/chart.png" alt="Patient trend chart">
  <span slot="fallback">Chart data unavailable.</span>
  <span slot="caption">Figure 2: 30-day vital sign trend.</span>
</hx-image>
```

## CSS Parts

| Part      | Element        | Description                                                                         |
| --------- | -------------- | ----------------------------------------------------------------------------------- |
| `base`    | `<img>`        | The inner image element. Target for `object-fit`, `object-position`, `filter`, etc. |
| `caption` | `<figcaption>` | The caption container. Only visible when caption slot content is present.           |

```css
/* Apply a CSS filter to the image */
hx-image::part(base) {
  filter: grayscale(100%);
  transition: filter 0.3s ease;
}

hx-image:hover::part(base) {
  filter: none;
}

/* Style the caption */
hx-image::part(caption) {
  font-style: italic;
  text-align: center;
}
```

| `--hx-image-caption-color`       | `var(--hx-color-neutral-600, #4b5563)` | Text color for the caption.                                                    |
| `--hx-image-caption-font-size`   | `0.875rem`                             | Font size for the caption.                                                     |
| `--hx-image-caption-padding`     | `0.5rem 0 0`                           | Padding for the caption element.                                               |

```css
hx-image {

  --hx-image-object-fit: contain;

/* Restyle captions site-wide */
hx-image {
  --hx-image-caption-color: var(--hx-color-neutral-700);
  --hx-image-caption-font-size: var(--hx-font-size-xs);
}
```

## Accessibility

`hx-image` enforces accessible image patterns at the component level, making it harder to ship inaccessible images by accident.

| Topic                         | Details                                                                                                                                                                                                                                                |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Alt text enforcement          | The `alt` property is `undefined` by default, not `""`. Informative images without `alt` will render with `alt=""` on the inner `<img>`, which browsers and axe-core treat as decorative. Always provide meaningful `alt` text for informative images. |
| Decorative images             | Use the `decorative` prop instead of `alt=""` to make decorative intent explicit. This sets `alt=""` and `role="presentation"` on the inner `<img>`, clearly communicating intent in markup and to CEM consumers.                                      |
| Error state announcement      | When image loading fails, the fallback container receives `role="alert"` and `aria-live="polite"`. This ensures screen readers announce the failure without requiring user interaction.                                                                |
| `role="presentation"` on img  | Applied automatically when `decorative` is `true` or when `alt=""` is explicitly provided. This hides the image from the accessibility tree entirely.                                                                                                  |
| Figure/figcaption structure   | The component renders `<figure>`/`<figcaption>` semantics. Captions are associated with the image via native HTML semantics, not ARIA.                                                                                                                 |
| WCAG 1.1.1 (Non-text Content) | Met. All informative images require `alt`. Decorative images must use `decorative` or `alt=""`.                                                                                                                                                        |
| WCAG 1.4.1 (Use of Color)     | Not applicable to `hx-image` directly. Ensure image content itself meets this criterion.                                                                                                                                                               |
| WCAG 2.1 AA                   | Verified. Zero axe-core violations with both informative and decorative image patterns.                                                                                                                                                                |

### Alt text guidelines

```html
<!-- Informative: describes what the image shows -->
<hx-image src="/photo.jpg" alt="Dr. Chen reviewing patient chart at nursing station"></hx-image>

<!-- Decorative: explicitly marked, not described -->
<hx-image src="/banner-bg.jpg" decorative></hx-image>

<!-- Functional: describes the action (if image acts as a link) -->
<!-- Wrap hx-image in an <a> and provide aria-label on the link -->
<a href="/dashboard" aria-label="Go to dashboard">
  <hx-image src="/icons/dashboard.png" decorative></hx-image>
</a>

<!-- Complex: reference a longer description elsewhere -->
<hx-image
  src="/charts/vitals.png"
  alt="Bar chart of patient vital signs over 7 days"
  aria-describedby="vitals-longdesc"
></hx-image>
<p id="vitals-longdesc" class="visually-hidden">
  Heart rate averaged 72 BPM. Blood pressure ranged from 118/78 to 128/84...
</p>
```

## Best Practices

**Always provide alt text for informative images.** In healthcare contexts, images may convey clinical information (scan results, anatomy diagrams, patient photos). Missing or inadequate alt text can constitute a patient safety concern for screen reader users.

**Use `decorative` explicitly.** Avoid using `alt=""` directly — use the `decorative` prop. This makes decorative intent visible in markup, searchable in CEM, and lintable by tooling.

**Lock aspect ratios to prevent layout shift.** Set `ratio` on all images whose dimensions are known at author-time. This eliminates Cumulative Layout Shift (CLS) during page load.

**Use `loading="eager"` only for above-the-fold images.** The default `loading="lazy"` is the right choice for content below the fold. Only use `eager` for the primary content image visible on initial page load (e.g., a hero or page header image).

**Provide a `fallback-src` for critical images.** In healthcare applications, images like patient photos or scan results may be critical to workflow. A low-resolution placeholder keeps the UI functional when the primary source fails.

**Use `srcset` and `sizes` for responsive images.** On bandwidth-constrained devices (mobile, rural connectivity), serving appropriately sized images reduces load time and data usage — a healthcare accessibility concern for users in underserved areas.

**Do not use `fit="fill"` without a fixed container.** `fill` stretches the image to fill the container without preserving aspect ratio. This typically distorts images. Use `cover` (crops) or `contain` (letterboxes) instead.

## Drupal Integration

`hx-image` is designed as a first-class Drupal component. The `srcset`/`sizes` attributes map directly to Drupal's Responsive Image module output.

```twig
{# my-module/templates/field--field-hero-image.html.twig #}
{% set img = items[0].content['#item'] %}

<hx-image
  src="{{ img.entity.uri.value | image_style('large') }}"
  srcset="{{ img | responsive_image_srcset('hero_responsive') }}"
  sizes="(max-width: 640px) 100vw, (max-width: 1024px) 80vw, 1200px"
  alt="{{ img.alt }}"
  ratio="16/9"
></hx-image>
```

For Drupal media entities with caption support:

```twig
{# With caption from media field #}
<hx-image
  src="{{ file_url(item.entity.field_media_image.entity.uri.value) }}"
  alt="{{ item.entity.field_media_image.alt }}"
  ratio="4/3"
  {% if item.entity.field_media_image.title %}
  data-title="{{ item.entity.field_media_image.title }}"
  {% endif %}
>
  {% if item.entity.field_caption.value %}
    <span slot="caption">{{ item.entity.field_caption.value }}</span>
  {% endif %}
</hx-image>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```
