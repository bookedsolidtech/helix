## Accessibility

`hx-switch` uses `role="switch"` (via a `<button>` element) to communicate the toggle state to assistive technologies, complying with the [ARIA switch pattern](https://www.w3.org/WAI/ARIA/apg/patterns/switch/).

### ARIA Roles and Attributes

- **Role**: `switch` (on the internal `<button>` element)
- **State**: `aria-checked="true|false"` reflects the `checked` property
- **Label**: The visible `label` prop or slotted content is associated via `aria-labelledby`
- **Error**: Error message is associated via `aria-describedby`
- **Help text**: Associated via `aria-describedby`

### Keyboard Interaction

| Key | Action |
|-----|--------|
| `Tab` | Move focus to switch |
| `Space` | Toggle switch on/off |
| `Enter` | Toggle switch on/off (some browser/AT combinations) |

### When to Use Switch vs Checkbox

- Use `hx-switch` for binary on/off settings that take **immediate effect** (e.g., enable dark mode, toggle feature flag).
- Use `hx-checkbox` for settings that take effect on **form submission** (e.g., agree to terms, select items).
- In healthcare, prefer `hx-checkbox` for clinical settings that require explicit confirmation before applying.

### Healthcare Compliance Notes

For switches that control safety-critical settings (e.g., enabling alerts, activating clinical modes):
- Provide confirmation dialogs before applying the change.
- Announce the new state clearly via `aria-live` region.
- Log state changes for audit purposes.

### Basic Switch Template

```twig

  name="{{ name }}"
  label="{{ label }}"
  value="{{ value|default('on') }}"

  {{ disabled ? 'disabled' : '' }}
  {{ required ? 'required' : '' }}
  hx-size="{{ size|default('md') }}"
  {{ error ? 'error="' ~ error ~ '"' : '' }}
  {{ help_text ? 'help-text="' ~ help_text ~ '"' : '' }}
></hx-switch>
```

### Drupal Form API Toggle

```twig
{# In a Drupal form — use as a styled boolean field #}
<hx-switch
  name="{{ element['#name'] }}"
  label="{{ element['#title'] }}"
  {{ element['#value'] ? 'checked' : '' }}
  {{ element['#disabled'] ? 'disabled' : '' }}
></hx-switch>
```

### Drupal Behavior: Immediate Effect

```javascript
(function (Drupal) {
  Drupal.behaviors.helixSwitch = {
    attach(context) {
      context.querySelectorAll('hx-switch[data-setting]').forEach((sw) => {
        sw.addEventListener('hx-change', async ({ detail }) => {
          const setting = sw.dataset.setting;
          try {
            await updateSetting(setting, detail.checked);
            Drupal.announce(`${sw.label} ${detail.checked ? 'enabled' : 'disabled'}`);
          } catch (err) {
            // Revert on failure
            sw.checked = !detail.checked;
            Drupal.announce('Failed to update setting. Please try again.');
          }
        });
      });
    }
  };
})(Drupal);
```

### Override Example

```css
/* Brand switch colors */
hx-switch {
  --hx-switch-track-checked-bg: var(--brand-color-success);
  --hx-switch-thumb-bg: #fff;
}

/* Danger switch for critical settings */
hx-switch.danger-switch[checked] {
  --hx-switch-track-checked-bg: var(--hx-color-error-600);
}
```

## Extending

### Listening to Toggle Events

```javascript
const sw = document.querySelector('hx-switch');

sw.addEventListener('hx-change', ({ detail }) => {
  console.log('Switch is now:', detail.checked ? 'on' : 'off');
  applyFeatureFlag(detail.checked);
});
```

### With Live Announcement

```javascript
const sw = document.querySelector('hx-switch[data-feature]');
const announcer = document.getElementById('sr-announcer');

sw.addEventListener('hx-change', ({ detail }) => {
  const feature = sw.dataset.feature;
  announcer.textContent = `${feature} has been ${detail.checked ? 'enabled' : 'disabled'}.`;
});
```
