## Overview

`hx-tabs` is a composite component consisting of three coordinated elements:

- **`hx-tabs`** — The container that manages state, keyboard navigation, and ARIA relationships.
- **`hx-tab`** — An individual tab trigger placed in the `tab` named slot. Renders as a `<button role="tab">`.
- **`hx-tab-panel`** — A content panel associated with a tab via matching `panel`/`name` attributes.

The component fully implements the [WAI-ARIA Tabs Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/) with roving `tabindex`, `aria-selected`, `aria-controls`/`aria-labelledby` cross-linking, and both automatic and manual keyboard activation modes.

**Use `hx-tabs` when:** users need to switch between a fixed set of related views without leaving the page — patient chart sections, record detail views, settings categories, or dashboard panes.

**Do not use tabs when:** content sections should be compared simultaneously (use a layout instead), there are more than 7–8 tabs (consider a `hx-side-nav` or pagination), or sections are independent pages (use navigation).

## Installation

```bash
# Full library
npm install @helix/library

import '@helix/library/components/hx-tabs';
```

## Basic Usage

```html
<hx-tabs label="Patient record sections">
  <hx-tab slot="tab" panel="overview">Overview</hx-tab>
  <hx-tab slot="tab" panel="medications">Medications</hx-tab>
  <hx-tab slot="tab" panel="history">History</hx-tab>

  <hx-tab-panel name="overview">
    <p>Overview content here.</p>
  </hx-tab-panel>
  <hx-tab-panel name="medications">
    <p>Medications content here.</p>
  </hx-tab-panel>
  <hx-tab-panel name="history">
    <p>History content here.</p>
  </hx-tab-panel>
</hx-tabs>
```

The `panel` attribute on `<hx-tab>` must match the `name` attribute on the corresponding `<hx-tab-panel>`. If no match is found, tabs and panels are paired by index order.

## Properties

### hx-tabs

| Property        | Attribute     | Type                         | Default        | Description                                                                                                                          |
| --------------- | ------------- | ---------------------------- | -------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| `orientation`   | `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | Layout orientation of the tablist. Horizontal renders tabs along the top; vertical renders them along the side.                      |
| `activation`    | `activation`  | `'automatic' \| 'manual'`    | `'automatic'`  | Keyboard activation mode. `'automatic'` activates a tab on arrow-key focus; `'manual'` requires Enter or Space to confirm selection. |
| `label`         | `label`       | `string`                     | `''`           | Accessible label for the tablist (`aria-label`). Provide a brief description of the tab group (e.g., `"Patient record sections"`).   |
| `selectedIndex` | —             | `number`                     | —              | Gets or sets the zero-based index of the currently selected tab. Setting this programmatically activates the tab at the given index. |

### hx-tab

| Property   | Attribute  | Type      | Default | Description                                                                                                       |
| ---------- | ---------- | --------- | ------- | ----------------------------------------------------------------------------------------------------------------- |
| `panel`    | `panel`    | `string`  | `''`    | The `name` of the `<hx-tab-panel>` this tab controls. Must match the `name` attribute on the corresponding panel. |
| `selected` | `selected` | `boolean` | `false` | Whether this tab is currently selected. Managed by the parent `<hx-tabs>`; avoid setting manually.                |
| `disabled` | `disabled` | `boolean` | `false` | Prevents the tab from being selected or reached via keyboard navigation.                                          |

### hx-tab-panel

| Property | Attribute | Type     | Default | Description                                                                                                                    |
| -------- | --------- | -------- | ------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `name`   | `name`    | `string` | `''`    | The name that corresponds to the `panel` attribute on the associated `<hx-tab>`. Must be unique within a `<hx-tabs>` instance. |

## Events

`hx-tabs` dispatches the following event:

| Event           | Detail Type                        | Description                                                                                                             |
| --------------- | ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| `hx-tab-change` | `{ tabId: string, index: number }` | Fired when the active tab changes. `tabId` is the `id` of the activated `<hx-tab>`; `index` is its zero-based position. |

```js
document.querySelector('hx-tabs').addEventListener('hx-tab-change', (e) => {
  console.log('Tab changed to index:', e.detail.index);
  console.log('Tab element id:', e.detail.tabId);
});
```

## CSS Parts

### hx-tabs

| Part      | Description                                                     |
| --------- | --------------------------------------------------------------- |
| `tablist` | The `<div role="tablist">` container wrapping all tab triggers. |
| `panels`  | The `<div>` container wrapping all `<hx-tab-panel>` elements.   |

### hx-tab

| Part     | Description                                                             |
| -------- | ----------------------------------------------------------------------- |
| `tab`    | The underlying `<button role="tab">` element.                           |
| `prefix` | The `<span>` container for content in the `prefix` slot (e.g., icons).  |
| `suffix` | The `<span>` container for content in the `suffix` slot (e.g., badges). |

### hx-tab-panel

| Part    | Description                                   |
| ------- | --------------------------------------------- |
| `panel` | The `<div>` content wrapper inside the panel. |

## Slots

### hx-tabs

| Slot        | Description                                                                                           |
| ----------- | ----------------------------------------------------------------------------------------------------- |
| `tab`       | Named slot for `<hx-tab>` elements. All tab triggers must be assigned to this slot.                   |
| _(default)_ | Default slot for `<hx-tab-panel>` elements. Panel order should match the tab order for index pairing. |

### hx-tab

| Slot        | Description                                            |
| ----------- | ------------------------------------------------------ |
| _(default)_ | The tab label text or content.                         |
| `prefix`    | Icon or element rendered before the label text.        |
| `suffix`    | Icon, badge, or element rendered after the label text. |

### hx-tab-panel

| Slot        | Description                                                      |
| ----------- | ---------------------------------------------------------------- |
| _(default)_ | The panel's content, shown when the corresponding tab is active. |

## Accessibility

`hx-tabs` implements the [WAI-ARIA Tabs Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/) (APG). All ARIA roles, states, and properties are managed automatically by the container.

| Topic                 | Details                                                                                                                                                        |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ARIA roles            | `role="tablist"` on the tablist container; `role="tab"` on each `<button>` inside `<hx-tab>`; `role="tabpanel"` on each `<hx-tab-panel>` host.                 |
| `aria-selected`       | Set to `'true'` on the active tab's inner button; `'false'` on all others.                                                                                     |
| `aria-controls`       | Set on the `role="tab"` button to reference the `id` of the corresponding `<hx-tab-panel>`.                                                                    |
| `aria-labelledby`     | Set on each `<hx-tab-panel>` to reference the `id` of its corresponding `<hx-tab>`, providing an accessible panel label.                                       |
| `aria-orientation`    | Set to `'horizontal'` or `'vertical'` on the tablist to communicate layout direction to screen readers.                                                        |
| `aria-label`          | Set on the tablist via the `label` attribute. Always provide a meaningful label (e.g., `"Patient record sections"`).                                           |
| `aria-disabled`       | Set to `'true'` on the inner button of disabled tabs. The disabled tab remains visually present and announced to screen readers.                               |
| Roving tabindex       | Only the active tab receives `tabindex="0"`; all others are `tabindex="-1"`. Arrow keys move focus between tabs without leaving the tablist.                   |
| Keyboard — tabs       | `ArrowRight` / `ArrowLeft` (horizontal) or `ArrowDown` / `ArrowUp` (vertical) move focus; `Home` jumps to first tab; `End` jumps to last tab.                  |
| Keyboard — activation | In `automatic` mode (default), arrow-key focus immediately activates the tab. In `manual` mode, press `Enter` or `Space` to activate the focused tab.          |
| Keyboard — panels     | `Tab` moves focus into the active panel; `Shift+Tab` returns focus to the active tab.                                                                          |
| Hidden panels         | Inactive panels have `hidden` set and `tabindex="-1"`, removing them from the accessibility tree and tab order until activated.                                |
| Focus ring            | Both tabs and focused panels display a visible focus ring meeting WCAG 2.1 AA minimum contrast. Customizable via `--hx-tabs-focus-ring-color`.                 |
| Reduced motion        | CSS transitions on tab button state changes are suppressed when `prefers-reduced-motion: reduce` is set.                                                       |
| WCAG                  | Meets WCAG 2.1 AA. Focus indicators are visible; color is not the sole conveyor of active state (font weight and indicator border also communicate selection). |

### Healthcare A11y Notes

In healthcare applications, always provide a meaningful `label` attribute describing the content domain (e.g., `"Patient Smith chart sections"`, not just `"Tabs"`). This ensures screen reader users navigating multiple tab groups on a complex clinical page can distinguish between them.

## Drupal Integration

```twig
{# my-module/templates/patient-record-tabs.html.twig #}
<hx-tabs
  label="{{ tabs_label|default('Record sections') }}"
  orientation="{{ orientation|default('horizontal') }}"
  activation="{{ activation|default('automatic') }}"
>
  {% for tab in tabs %}
    <hx-tab
      slot="tab"
      panel="{{ tab.panel_name }}"
      {% if tab.disabled %}disabled{% endif %}
    >
      {{ tab.label }}
    </hx-tab>
  {% endfor %}

  {% for panel in panels %}
    <hx-tab-panel name="{{ panel.name }}">
      {{ panel.content|raw }}
    </hx-tab-panel>
  {% endfor %}
</hx-tabs>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

Listen for tab changes in Drupal behaviors using `once()` to prevent duplicate listeners on AJAX:

```javascript
Drupal.behaviors.helixTabs = {
  attach(context) {
    // once() is a Drupal core utility — prevents duplicate event binding during AJAX attach cycles
    once('helixTabs', 'hx-tabs', context).forEach((el) => {
      el.addEventListener('hx-tab-change', (e) => {
        console.log('Active tab changed:', e.detail.tabId, 'at index', e.detail.index);
        // Example: update URL hash to preserve selected tab on reload
        history.replaceState(null, '', `#${e.detail.tabId}`);
      });
    });
  },
};
```

## Programmatic Control

```js
const tabs = document.querySelector('hx-tabs');

// Get the currently active tab index (zero-based)
console.log(tabs.selectedIndex); // e.g., 0

// Programmatically activate the second tab
tabs.selectedIndex = 1;

// Listen for changes
tabs.addEventListener('hx-tab-change', (e) => {
  console.log('Changed to:', e.detail.index, e.detail.tabId);
});
```

## Standalone HTML Example

Copy-paste into a `.html` file and open in any browser — no build tool required:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>hx-tabs example</title>
    <!-- @helix/library is a private package — install via npm workspace, not CDN -->
    <!-- In your project: import '@helix/library'; in your bundler entry point -->
  </head>
  <body style="font-family: sans-serif; padding: 2rem; max-width: 800px; margin: 0 auto;">
    <h1>Patient Record</h1>

    <hx-tabs id="patient-tabs" label="Patient record sections">
      <hx-tab slot="tab" panel="overview">Overview</hx-tab>
      <hx-tab slot="tab" panel="medications">Medications</hx-tab>
      <hx-tab slot="tab" panel="history">History</hx-tab>
      <hx-tab slot="tab" panel="notes" disabled>Notes (locked)</hx-tab>

      <hx-tab-panel name="overview">
        <h2>Patient Overview</h2>
        <p>Jane Doe, DOB: 1975-04-12. Primary care provider: Dr. Smith.</p>
      </hx-tab-panel>

      <hx-tab-panel name="medications">
        <h2>Current Medications</h2>
        <ul>
          <li>Lisinopril 10mg — once daily</li>
          <li>Metformin 500mg — twice daily</li>
          <li>Atorvastatin 20mg — at bedtime</li>
        </ul>
      </hx-tab-panel>

      <hx-tab-panel name="history">
        <h2>Medical History</h2>
        <p>Hypertension (2021), Type 2 Diabetes (2020), Appendectomy (2018).</p>
      </hx-tab-panel>

      <hx-tab-panel name="notes">
        <h2>Clinical Notes</h2>
        <p>Access restricted — contact your administrator.</p>
      </hx-tab-panel>
    </hx-tabs>

    <script>
      const tabs = document.getElementById('patient-tabs');

      tabs.addEventListener('hx-tab-change', (e) => {
        console.log('Tab changed to index:', e.detail.index, '— id:', e.detail.tabId);
      });
    </script>
  </body>
</html>
```
