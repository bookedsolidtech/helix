## Overview

`hx-drawer` renders a slide-in panel from any edge of the viewport (start, end, top, or bottom). It implements the ARIA dialog pattern: `role="dialog"`, `aria-modal="true"`, full focus trapping, Escape-to-close, and automatic focus return to the trigger element on close. Body scroll is locked while the drawer is open to prevent accidental background interaction.

**Use `hx-drawer` when:** you need a secondary panel for contextual detail, forms, navigation trees, or filtered search results that overlay the primary content without fully leaving the page. Common healthcare use cases include patient detail panels, medication order forms, and care team navigation drawers.

**Do not use** `hx-drawer` for confirmation dialogs that require an immediate decision — use a modal dialog instead. Do not use it for primary page navigation if the destination is a separate page — use standard anchor links.

## Basic Usage

```html
<hx-drawer id="my-drawer" placement="end" size="md">
  <span slot="label">Drawer Title</span>

  <p>Body content goes here — forms, lists, or any secondary content.</p>

  <button slot="footer" onclick="document.getElementById('my-drawer').hide()">Cancel</button>
  <button slot="footer" onclick="handleSave()">Save</button>
</hx-drawer>

<!-- Trigger -->
<button onclick="document.getElementById('my-drawer').show()">Open Drawer</button>
```

## Properties

| Property    | Attribute   | Type                                       | Default | Description                                                                                 |
| ----------- | ----------- | ------------------------------------------ | ------- | ------------------------------------------------------------------------------------------- |
| `open`      | `open`      | `boolean`                                  | `false` | Controls whether the drawer is open. Reflects to the host element attribute.                |
| `placement` | `placement` | `'start' \| 'end' \| 'top' \| 'bottom'`    | `'end'` | Which viewport edge the drawer slides in from.                                              |
| `size`      | `size`      | `'sm' \| 'md' \| 'lg' \| 'full' \| string` | `'md'`  | Panel width (or height for top/bottom). Accepts preset keys or any valid CSS length string. |
| `contained` | `contained` | `boolean`                                  | `false` | When true, the drawer is scoped to its positioned parent rather than the viewport.          |
| `noHeader`  | `no-header` | `boolean`                                  | `false` | When true, hides the entire header region (title, header-actions, close button).            |
| `noFooter`  | `no-footer` | `boolean`                                  | `false` | When true, hides the footer slot region.                                                    |
| `label`     | `label`     | `string`                                   | `''`    | Accessible label string for `aria-label` when the `label` slot is not used.                 |

## Events

| Event              | Detail Type         | Description                                                                                                                                       |
| ------------------ | ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `hx-show`          | `void`              | Fired immediately when the drawer begins to open (before animation).                                                                              |
| `hx-after-show`    | `void`              | Fired after the open animation completes (300ms by default, 0ms with reduced-motion).                                                             |
| `hx-hide`          | `void`              | Fired immediately when the drawer begins to close (before animation).                                                                             |
| `hx-after-hide`    | `void`              | Fired after the close animation completes. Focus is returned to the trigger at this point.                                                        |
| `hx-initial-focus` | `void` (cancelable) | Fired when the drawer is about to set initial focus. Call `event.preventDefault()` to override the default focus behavior and set focus manually. |

## Slots

| Slot             | Description                                                                                                 |
| ---------------- | ----------------------------------------------------------------------------------------------------------- |
| _(default)_      | The scrollable body region. Accepts any content: text, forms, lists, or nested components.                  |
| `label`          | The drawer title text. When populated, `aria-labelledby` is applied automatically pointing to this element. |
| `header-actions` | Optional action buttons rendered in the header row alongside the close button.                              |
| `footer`         | Action buttons or footer content (e.g., Cancel / Save). Automatically hidden when empty.                    |

## CSS Parts

| Part           | Description                                                                                     |
| -------------- | ----------------------------------------------------------------------------------------------- |
| `overlay`      | The full-screen overlay container that holds the backdrop and panel. Fixed-position by default. |
| `panel`        | The drawer panel itself. Use to override background, shadow, or border-radius.                  |
| `header`       | The header region containing the title, header-actions slot, and close button.                  |
| `title`        | The `<p>` element wrapping the `label` slot. Use to override typography.                        |
| `close-button` | The built-in close button. Use to restyle or resize the icon button.                            |
| `body`         | The scrollable body region wrapping the default slot.                                           |
| `footer`       | The footer region wrapping the `footer` slot.                                                   |

## Accessibility

| Topic             | Details                                                                                                                                                                                                                                                                              |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| ARIA role         | `role="dialog"` on the overlay container. This is the correct ARIA landmark for all modal and non-modal drawers per the ARIA Authoring Practices Guide (APG).                                                                                                                        |
| `aria-modal`      | `aria-modal="true"` is set to signal that background content is inert. Additionally, all sibling elements in `document.body` receive `aria-hidden="true"` when the drawer opens, and are restored on close. This dual approach ensures compatibility with NVDA, JAWS, and VoiceOver. |
| Accessible name   | When the `label` slot is populated, `aria-labelledby` points to the title element. When the slot is empty, `aria-label` falls back to the `label` property value, or "Drawer" if neither is provided. The dialog always has an accessible name.                                      |
| Focus trap        | `Tab` and `Shift+Tab` cycle focus within the drawer panel — focus cannot escape to background content. Focus detection uses `document.activeElement` to correctly handle slotted (light DOM) elements.                                                                               |
| Initial focus     | When the drawer opens, focus moves to the first focusable element inside the panel. Listen for `hx-initial-focus` and call `event.preventDefault()` to customize this behavior.                                                                                                      |
| Focus restoration | When the drawer closes, focus returns to the element that was focused before the drawer opened (the trigger element).                                                                                                                                                                |
| Escape key        | Pressing `Escape` closes the drawer from any focused element — including elements outside the Shadow DOM. The listener is registered on `document` immediately when the drawer opens (C-PATTERN-06).                                                                                 |
| Body scroll lock  | `document.body.style.overflow = 'hidden'` is set on open and restored on close. Not applied in `contained` mode.                                                                                                                                                                     |
| Close button      | The built-in close button has `aria-label="Close drawer"` and is keyboard-focusable.                                                                                                                                                                                                 |
| Reduced motion    | Transitions use CSS `transition` with a 300ms default. When `prefers-reduced-motion: reduce` is set, transitions are disabled in CSS and the JavaScript animation duration resolves to 0ms.                                                                                          |
| WCAG 2.1 AA       | Verified with axe-core: zero violations in both open (with label) and closed states.                                                                                                                                                                                                 |

## Keyboard Navigation

| Key               | Behavior                                                                               |
| ----------------- | -------------------------------------------------------------------------------------- |
| `Tab`             | Moves focus to the next focusable element inside the drawer. Wraps from last to first. |
| `Shift + Tab`     | Moves focus to the previous focusable element. Wraps from first to last.               |
| `Escape`          | Closes the drawer. Focus returns to the trigger element.                               |
| `Enter` / `Space` | Activates the focused button (close button, footer actions, etc.).                     |

## Drupal Integration

Use `hx-drawer` in Twig templates after registering the Helix library:

```twig
{# my-module/templates/patient-detail.html.twig #}
<button
  data-hx-drawer-trigger
  data-hx-drawer-target="patient-{{ patient.id }}-drawer"
>
  View Patient Details
</button>

<hx-drawer
  id="patient-{{ patient.id }}-drawer"
  placement="end"
  size="lg"
  label="{{ 'Patient Details'|t }}"
>
  <span slot="label">{{ patient.full_name|t }}</span>

  <dl>
    <dt>{{ 'MRN'|t }}</dt>
    <dd>{{ patient.mrn|e('html') }}</dd>
    <dt>{{ 'Room'|t }}</dt>
    <dd>{{ patient.room|e('html') }}</dd>
  </dl>

  <button slot="footer" data-hx-drawer-close>{{ 'Close'|t }}</button>
</hx-drawer>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

Wire open/close using Drupal behaviors:

```javascript
Drupal.behaviors.helixDrawer = {
  attach(context) {
    // Open trigger
    context.querySelectorAll('[data-hx-drawer-trigger]').forEach((trigger) => {
      const targetId = trigger.dataset.hxDrawerTarget;
      const drawer = document.getElementById(targetId);
      if (!drawer) return;

      trigger.addEventListener('click', () => {
        drawer.show();
      });
    });

    // Close buttons inside the drawer
    context.querySelectorAll('[data-hx-drawer-close]').forEach((btn) => {
      const drawer = btn.closest('hx-drawer');
      if (!drawer) return;

      btn.addEventListener('click', () => {
        drawer.hide();
      });
    });

    // Optional: react to events
    context.querySelectorAll('hx-drawer').forEach((drawer) => {
      drawer.addEventListener('hx-show', () => {
        // e.g., load content dynamically on open
      });

      drawer.addEventListener('hx-after-hide', () => {
        // e.g., reset form state after close
      });
    });
  },
};
```
