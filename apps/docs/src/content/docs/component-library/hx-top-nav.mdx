## Overview

`hx-top-nav` is the site-level navigation bar for Helix applications. It renders a `<header>` landmark containing a `<nav>` element with a three-zone layout: a **logo** area on the left, a **primary navigation** area in the center, and an **actions** area on the right.

On narrow viewports (below 768 px), the center and right zones collapse behind a hamburger toggle button. Pressing the button opens a vertical stack of nav items and actions. On desktop (768 px and above), the hamburger is hidden and the full navigation is shown inline.

All content is slot-driven — `hx-top-nav` imposes layout and accessibility structure; consumers supply the links, branding, and utility controls. The component is ready for Drupal Twig consumption with no modifications.

**Use `hx-top-nav` when:** you need a top-of-page navigation bar with responsive collapse, sticky positioning, and accessible landmark structure for a healthcare portal, staff tool, or patient-facing application.

**Do not place a `<nav>` element** inside any slot — the component already renders a `<nav>` landmark internally. Use a `<div>` or bare links in the default slot.

## Installation

```bash
# Full library
npm install @helixui/library

# Or import only this component (tree-shaking friendly)
import '@helixui/library/components/hx-top-nav';
```

## Basic Usage

```html
<hx-top-nav label="Site Navigation">
  <a slot="logo" href="/">My App</a>

  <!-- Use bare links or <div style="display: contents;"> — NOT <nav> -->
  <a href="/dashboard">Dashboard</a>
  <a href="/patients">Patients</a>
  <a href="/schedule">Schedule</a>

  <div slot="actions">
    <button type="button" aria-label="User profile">Profile</button>
  </div>
</hx-top-nav>
```

## Properties

| Property | Attribute | Type      | Default             | Description                                                                                                            |
| -------- | --------- | --------- | ------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `sticky` | `sticky`  | `boolean` | `false`             | When true, the navigation bar sticks to the top of the viewport during scroll. Reflects as attribute.                  |
| `label`  | `label`   | `string`  | `'Site Navigation'` | Accessible label applied to the inner `<nav>` element via `aria-label`. Describes the landmark to screen reader users. |

## Events

| Event              | Detail Type         | Description                                                                                                                              |
| ------------------ | ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `hx-mobile-toggle` | `{ open: boolean }` | Dispatched when the hamburger button is toggled. `detail.open` is `true` when the menu opens, `false` when it closes. Bubbles, composed. |

```js
document.querySelector('hx-top-nav').addEventListener('hx-mobile-toggle', (e) => {
  console.log('Mobile menu is now:', e.detail.open ? 'open' : 'closed');
});
```

## CSS Parts

| Part            | Description                                    |
| --------------- | ---------------------------------------------- |
| `header`        | The outer `<header>` landmark element.         |
| `nav`           | The `<nav>` element inside the header.         |
| `logo`          | The logo slot container `<div>`.               |
| `menu`          | The primary navigation slot container `<div>`. |
| `actions`       | The actions slot container `<div>`.            |
| `mobile-toggle` | The hamburger toggle `<button>`.               |

```css
/* Example: custom brand bar */
hx-top-nav::part(nav) {
  background: #0f172a;
  border-bottom-color: #334155;
}

hx-top-nav::part(mobile-toggle) {
  border-radius: 0.5rem;
}
```

## Slots

| Slot        | Description                                                                                                                                                             |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `logo`      | Brand area rendered on the left side. Typically an `<a>` wrapping a logo image or styled text.                                                                          |
| _(default)_ | Primary navigation items rendered in the center (desktop) or collapsible panel (mobile). Use bare links or `<div style="display: contents;">` — do NOT wrap in `<nav>`. |
| `actions`   | Utility area on the right side. Suitable for search inputs, user profile menus, notification buttons, or sign-out links.                                                |

## Accessibility

`hx-top-nav` implements the [WAI-ARIA Navigation Landmark](https://www.w3.org/TR/wai-aria-1.2/#navigation) pattern and the [WAI-ARIA Button Pattern](https://www.w3.org/WAI/ARIA/apg/patterns/button/) for the mobile toggle.

| Topic               | Details                                                                                                                                                                       |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Navigation landmark | The inner `<nav>` element creates a navigation landmark, discoverable by screen reader quick navigation commands.                                                             |
| `aria-label`        | The `label` property (default `"Site Navigation"`) is applied to the `<nav>` via `aria-label`. Always set a meaningful label to distinguish multiple nav landmarks on a page. |
| `<header>` landmark | The outer `<header>` element creates a banner landmark when used outside of `<article>`, `<aside>`, `<main>`, `<nav>`, or `<section>`.                                        |
| Mobile toggle       | The hamburger `<button>` carries `aria-expanded` (reflects current open state), `aria-controls` (references `id="nav-menu"`), and `aria-label="Toggle navigation"`.           |
| `aria-expanded`     | Set to `"true"` when the mobile menu is open; `"false"` when collapsed. Screen readers announce the state change on button activation.                                        |
| `aria-current`      | Not managed internally — mark the active page link with `aria-current="page"` in the consuming template (Drupal `in_active_trail`, etc.).                                     |
| Keyboard — toggle   | The hamburger toggle is a native `<button>` — activatable with `Enter` or `Space`.                                                                                            |
| Keyboard — escape   | Pressing `Escape` when the mobile menu is open closes the menu and returns focus to the toggle button.                                                                        |
| Focus management    | When the mobile menu opens, focus moves to the first interactive element in the default slot.                                                                                 |
| Touch target        | The hamburger button meets WCAG 2.5.5 (44×44 px minimum target size): 0.75 rem padding on all sides around a 24 px icon yields a 48×48 px touch target.                       |
| Focus ring          | The hamburger button shows a visible focus ring via `:focus-visible` using `--hx-focus-ring-color`. All slotted interactive elements retain their own focus rings.            |
| Reduced motion      | `@media (prefers-reduced-motion: reduce)` disables the mobile menu open animation.                                                                                            |
| WCAG                | WCAG 2.1 AA compliant. Zero axe-core violations in default, sticky, and mobile-open states.                                                                                   |

### Healthcare A11y Notes

Always provide a meaningful `label` attribute. On complex clinical pages with multiple navigation landmarks (e.g., `hx-top-nav` + `hx-side-nav`), each landmark must have a distinct `aria-label` so screen reader users can distinguish between them.

Use `aria-current="page"` on the currently active link. In Drupal, use `in_active_trail` from the menu system:

```twig
<a href="{{ item.url }}"
   {% if item.in_active_trail %}aria-current="page"{% endif %}>
  {{ item.title }}
</a>
```

## Keyboard Navigation

| Key             | Behavior                                                                                         |
| --------------- | ------------------------------------------------------------------------------------------------ |
| `Tab`           | Moves focus through focusable elements in the nav bar (logo link, toggle button, slotted items). |
| `Enter`/`Space` | Activates the hamburger toggle button when it has focus (mobile viewports).                      |
| `Escape`        | Closes the mobile menu and returns focus to the hamburger toggle button.                         |

## Drupal Integration

```twig
{# my-module/templates/hx-top-nav.html.twig #}
<hx-top-nav
  label="{{ nav_label|default('Site Navigation') }}"
  {% if sticky %}sticky{% endif %}
>
  <a slot="logo" href="{{ path('<front>') }}" class="logo-link">
    {{ site_name }}
  </a>

  {# Use bare links or <div style="display: contents;"> — NOT <nav> #}
  {% for item in main_menu_items %}
    <a
      href="{{ item.url }}"
      {% if item.in_active_trail %}aria-current="page"{% endif %}
      class="nav-link"
    >{{ item.title }}</a>
  {% endfor %}

  <div slot="actions">
    <a href="{{ path('user.page') }}">{{ 'My Account'|t }}</a>
    <a href="{{ path('user.logout') }}">{{ 'Sign Out'|t }}</a>
  </div>
</hx-top-nav>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

Listen for mobile toggle events in Drupal behaviors:

```javascript
Drupal.behaviors.helixTopNav = {
  attach(context) {
    // once() prevents duplicate event binding during AJAX attach cycles
    once('helixTopNav', 'hx-top-nav', context).forEach((el) => {
      el.addEventListener('hx-mobile-toggle', (e) => {
        console.log('Mobile menu state:', e.detail.open ? 'open' : 'closed');
        // Example: update body class for CSS adjustments
        document.body.classList.toggle('mobile-nav-open', e.detail.open);
      });
    });
  },
};
```

## Standalone HTML Example

Copy-paste into a `.html` file and open in any browser — no build tool required:

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>hx-top-nav example</title>
    <!-- @helixui/library is a private package — install via npm workspace, not CDN -->
    <!-- In your project: import '@helixui/library'; in your bundler entry point -->
  </head>
  <body style="font-family: sans-serif; margin: 0; padding: 0;">
    <hx-top-nav id="site-nav" sticky label="Hospital Staff Navigation">
      <a
        slot="logo"
        href="/"
        style="font-weight: 700; color: inherit; text-decoration: none; font-size: 1.125rem;"
      >
        Memorial Health
      </a>

      <!-- Bare links or <div style="display: contents;"> — NOT <nav> -->
      <a
        href="/dashboard"
        aria-current="page"
        style="color: inherit; text-decoration: none; padding: 0.5rem 0.75rem; font-weight: 600;"
        >Dashboard</a
      >
      <a href="/patients" style="color: inherit; text-decoration: none; padding: 0.5rem 0.75rem;"
        >Patients</a
      >
      <a href="/schedule" style="color: inherit; text-decoration: none; padding: 0.5rem 0.75rem;"
        >Schedule</a
      >
      <a href="/lab-results" style="color: inherit; text-decoration: none; padding: 0.5rem 0.75rem;"
        >Lab Results</a
      >

      <div slot="actions" style="display: flex; align-items: center; gap: 0.75rem;">
        <input
          type="search"
          placeholder="Search patients..."
          aria-label="Search patients"
          style="padding: 0.375rem 0.75rem; border: 1px solid #ced4da; border-radius: 0.375rem; font-size: 0.875rem; width: 14rem;"
        />
        <button
          type="button"
          style="background: none; border: 1px solid currentColor; border-radius: 0.375rem; cursor: pointer; padding: 0.375rem 0.875rem; font-size: 0.875rem;"
        >
          Sign Out
        </button>
      </div>
    </hx-top-nav>

    <main style="padding: 2rem;">
      <h1>Dashboard</h1>
      <p>Clinical workflow content goes here.</p>
    </main>

    <script>
      const nav = document.getElementById('site-nav');

      nav.addEventListener('hx-mobile-toggle', (e) => {
        console.log('Mobile menu is now:', e.detail.open ? 'open' : 'closed');
      });
    </script>
  </body>
</html>
```
