## Overview

`hx-textarea` is a form-associated multi-line text input component for enterprise healthcare interfaces. It provides built-in label rendering, error messaging, help text, character counting, and auto-grow resize — all with full WCAG 2.1 AA accessibility via `ElementInternals` form association, `aria-invalid` error state, and `aria-describedby` linkage to error and help text regions.

**Use `hx-textarea` when:** you need a multi-line text field for clinical notes, patient comments, free-form data entry, or any form field that benefits from a larger input area with constraint validation and screen reader support.

**Use `hx-text-input` instead when:** the data is a single line (names, IDs, dates, phone numbers). `hx-textarea` is specifically for multi-line content.

**Drupal note:** All three slots (`label`, `error`, `help-text`) are designed to accept Drupal Form API-rendered markup. Slotted content works independently of the matching property — you do not need to set both.

## Basic Usage

Minimal HTML — no build tool required:

```html
<!-- Basic textarea with label -->
<hx-textarea label="Clinical Notes" placeholder="Enter notes..."></hx-textarea>

<!-- Required field with help text -->
<hx-textarea
  label="Diagnosis"
  required
  help-text="Include ICD-10 codes where applicable."
></hx-textarea>

<!-- Error state -->
<hx-textarea label="Notes" error="This field is required."></hx-textarea>

<!-- With character count and maxlength -->
<hx-textarea
  label="Short bio"
  maxlength="200"
  show-count
  help-text="Max 200 characters."
></hx-textarea>

<!-- Auto-grow (expands as user types) -->
<hx-textarea label="Free text" resize="auto" rows="2"></hx-textarea>

<!-- Disabled -->
<hx-textarea label="Read-only notes" value="Cannot be changed." disabled></hx-textarea>

<!-- Accessible name without visible label -->
<hx-textarea aria-label="Patient notes"></hx-textarea>
```

Set properties via JavaScript for dynamic control:

```javascript
const textarea = document.querySelector('hx-textarea');

// Listen for input events
textarea.addEventListener('hx-input', (event) => {
  console.log('Current value:', event.detail.value);
});

// Listen for change events (fires on blur after value changes)
textarea.addEventListener('hx-change', (event) => {
  console.log('Final value:', event.detail.value);
});

// Programmatic value update
textarea.value = 'Pre-populated text';

// Set error state dynamically
textarea.error = 'Value must be at least 50 characters.';

// Focus the textarea
textarea.focus();

// Select all text
textarea.select();

// Form validation
if (!textarea.checkValidity()) {
  textarea.reportValidity();
}
```

## Properties

| Property      | Attribute     | Type                                       | Default      | Description                                                                                                                       |
| ------------- | ------------- | ------------------------------------------ | ------------ | --------------------------------------------------------------------------------------------------------------------------------- |
| `label`       | `label`       | `string`                                   | `''`         | Visible label text. Renders a `<label>` associated to the textarea via `for`/`id`. Overridden by the `label` slot.                |
| `placeholder` | `placeholder` | `string`                                   | `''`         | Placeholder text shown when the textarea is empty.                                                                                |
| `value`       | `value`       | `string`                                   | `''`         | Current value of the textarea. Kept in sync with the native `<textarea>` via `live()`.                                            |
| `required`    | `required`    | `boolean`                                  | `false`      | Marks the field as required. Shows a `*` marker next to the label. Sets `required` on the native textarea. Reflects to attribute. |
| `disabled`    | `disabled`    | `boolean`                                  | `false`      | Disables the textarea. Suppresses interaction and applies reduced opacity to the host. Reflects to attribute.                     |
| `error`       | `error`       | `string`                                   | `''`         | Error message to display. Triggers `aria-invalid="true"` on the textarea and renders an error div with `role="alert"`.            |
| `helpText`    | `help-text`   | `string`                                   | `''`         | Help text displayed below the textarea. Hidden when `error` is set. Overridden by the `help-text` slot.                           |
| `name`        | `name`        | `string`                                   | `''`         | Name for form submission. Passed to the native textarea.                                                                          |
| `rows`        | `rows`        | `number`                                   | `4`          | Number of visible text rows.                                                                                                      |
| `maxlength`   | `maxlength`   | `number \| undefined`                      | `undefined`  | Maximum character count. Enforced by form validation (`tooLong`) and displayed in the character counter when `show-count` is set. |
| `resize`      | `resize`      | `'none' \| 'vertical' \| 'both' \| 'auto'` | `'vertical'` | Resize behavior. `'auto'` grows with content. Reflects to attribute.                                                              |
| `showCount`   | `show-count`  | `boolean`                                  | `false`      | Shows a character counter below the textarea. Displays `count / maxlength` when `maxlength` is set, or just `count` otherwise.    |
| `ariaLabel`   | `aria-label`  | `string \| null`                           | `null`       | Accessible name for screen readers when no visible label is present. Set directly on the native `<textarea>`.                     |

## Events

| Event       | Detail Type         | Bubbles | Composed | Description                                                                       |
| ----------- | ------------------- | ------- | -------- | --------------------------------------------------------------------------------- |
| `hx-input`  | `{ value: string }` | `true`  | `true`   | Dispatched on every keystroke as the user types. `detail.value` is the new value. |
| `hx-change` | `{ value: string }` | `true`  | `true`   | Dispatched when the textarea loses focus after its value changed.                 |

```javascript
const textarea = document.querySelector('hx-textarea');

// Real-time input tracking (every keystroke)
textarea.addEventListener('hx-input', (e) => {
  console.log('Typing:', e.detail.value);
});

// Committed value (on blur after change)
textarea.addEventListener('hx-change', (e) => {
  saveToServer(e.detail.value);
});
```

## CSS Parts

| Part               | Description                                                              |
| ------------------ | ------------------------------------------------------------------------ |
| `field`            | The outer `<div>` container wrapping all field elements.                 |
| `label`            | The `<label>` element (only present when `label` property is set).       |
| `textarea-wrapper` | The wrapper `<div>` around the native textarea that renders the border.  |
| `textarea`         | The native `<textarea>` element.                                         |
| `counter`          | The character count display (only present when `show-count` is set).     |
| `help-text`        | The help text container (only present when help text is available).      |
| `error`            | The error message container with `role="alert"` (only present on error). |

Use CSS parts to style internals not exposed by custom properties:

```css
/* Adjust the textarea padding */
hx-textarea::part(textarea) {
  padding: 0.75rem 1rem;
}

/* Style the character counter */
hx-textarea::part(counter) {
  font-style: italic;
}

/* Style the error message */
hx-textarea::part(error) {
  font-weight: 600;
}
```

## Slots

| Slot        | Description                                                                                                                                                                              |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `label`     | Custom label content. Overrides the `label` property. Use for Drupal Form API rendered labels or rich label markup. The slotted element receives a generated `id` for `aria-labelledby`. |
| `help-text` | Custom help text. Overrides the `help-text` property. Renders independently — the `help-text` property does not need to be set. Hidden when an error is active.                          |
| `error`     | Custom error content. Overrides the `error` property. Renders the error container (with `role="alert"`) when slotted content is present. Use for Drupal Form API errors.                 |

```html
<!-- Slotted label with rich markup -->
<hx-textarea>
  <label slot="label"> Diagnosis <abbr title="required" aria-hidden="true">*</abbr> </label>
</hx-textarea>

<!-- Slotted help text (no property needed) -->
<hx-textarea>
  <span slot="help-text">Include ICD-10 codes where applicable.</span>
</hx-textarea>

<!-- Slotted error (no property needed) -->
<hx-textarea>
  <span slot="error">This field is required.</span>
</hx-textarea>
```

## Accessibility

`hx-textarea` is built for WCAG 2.1 AA compliance in healthcare contexts where accurate communication to assistive technology users is mandatory.

| Topic              | Details                                                                                                                                                                                                                         |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Role               | The native `<textarea>` element has an implicit `textbox` role with `aria-multiline="true"`. No additional ARIA role is set.                                                                                                    |
| Label association  | When the `label` property is set, a `<label for>` association is created via matching `id`/`for` attributes. When the `label` slot is used, `aria-labelledby` is set on the textarea pointing to the slotted element's `id`.    |
| `aria-label`       | The `ariaLabel` property sets `aria-label` directly on the native textarea for cases without a visible label.                                                                                                                   |
| Error state        | `aria-invalid="true"` is set on the textarea when an error is present (via property or slot). The error container uses `role="alert"` for immediate announcement without a conflicting `aria-live` attribute.                   |
| `aria-describedby` | The textarea's `aria-describedby` is automatically computed to reference: the error container ID (when error is active), the help text container ID (when help text is visible), and the counter ID (when `show-count` is set). |
| Required           | The native `required` attribute is set on the `<textarea>`. `aria-required` is intentionally not added (redundant per HTML-AAM specification).                                                                                  |
| Character counter  | The counter has `aria-live="polite"` to announce count updates as the user types. It is also included in `aria-describedby` so screen readers can discover it when focused.                                                     |
| Focus indicator    | The textarea wrapper renders a visible focus ring using `--hx-focus-ring-color`, `--hx-focus-ring-width`, and `--hx-focus-ring-opacity` design tokens.                                                                          |
| WCAG               | Meets WCAG 2.1 AA. SC 1.3.1 (Info and Relationships), SC 1.3.5 (Identify Input Purpose), SC 4.1.2 (Name, Role, Value), SC 4.1.3 (Status Messages) all satisfied. Zero axe-core violations across all states.                    |

### Keyboard Navigation

| Key                 | Action                                                                         |
| ------------------- | ------------------------------------------------------------------------------ |
| `Tab`               | Moves focus to the textarea.                                                   |
| Any key             | Standard browser textarea keyboard behavior (typing, navigation within text).  |
| `Tab` (in textarea) | Moves focus out of the textarea to the next focusable element in the document. |

### Screen Reader Guidance

```html
<!-- With label property: announced as "Clinical Notes, text area" -->
<hx-textarea label="Clinical Notes"></hx-textarea>

<!-- With error: announced as "Clinical Notes, text area, Required field" -->
```

## Drupal Integration

Use `hx-textarea` in a Twig template after registering the library.

```twig
{# my-module/templates/my-template.html.twig #}

{# Basic textarea #}
<hx-textarea
  label="{{ label }}"
  placeholder="{{ placeholder|default('') }}"
  {% if required %}required{% endif %}
>
</hx-textarea>

{# With Drupal Form API help text via slot (no help-text property needed) #}
<hx-textarea
  label="{{ label }}"
  {% if required %}required{% endif %}
>
  {% if description %}
    <span slot="help-text">{{ description }}</span>
  {% endif %}
</hx-textarea>

{# With Drupal Form API error messages via slot #}
<hx-textarea
  label="{{ label }}"
  {% if required %}required{% endif %}
>
  {% if errors %}
    <span slot="error">{{ errors }}</span>
  {% endif %}
  {% if description and not errors %}
    <span slot="help-text">{{ description }}</span>
  {% endif %}
</hx-textarea>

{# With character count for clinical note fields #}
<hx-textarea
  label="{{ label }}"
  name="{{ field_name }}"
  maxlength="{{ maxlength|default(500) }}"
  show-count
  resize="auto"
  rows="4"
  {% if required %}required{% endif %}
  {% if disabled %}disabled{% endif %}
>
</hx-textarea>

{# With fully custom label slot for rich markup #}
<hx-textarea>
  <label slot="label">
    {{ label }}
    {% if required %}<abbr title="{{ 'Required'|t }}" aria-hidden="true">*</abbr>{% endif %}
  </label>
</hx-textarea>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/helix.min.js: { minified: true }
```

Listen for events in Drupal behaviors using `once()` to prevent duplicate listeners on AJAX:

```javascript
Drupal.behaviors.helixTextarea = {
  attach(context) {
    // once() is a Drupal core utility — prevents duplicate event binding during AJAX attach cycles
    once('helixTextareaChange', 'hx-textarea', context).forEach((el) => {
      el.addEventListener('hx-change', (e) => {
        // Save the value or update form state
        const fieldName = el.getAttribute('name');
        console.log(`${fieldName} updated:`, e.detail.value);
      });
    });
  },
};
```
