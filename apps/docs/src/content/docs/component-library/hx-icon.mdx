## Overview

`hx-icon` is an SVG icon component that supports two render modes: **sprite mode** (recommended) and **inline fetch mode**. It handles decorative vs. informative icon semantics automatically — when a `label` is provided the icon is announced as an image; when no label is present `aria-hidden="true"` is applied so assistive technology skips it entirely.

**Sprite mode** (recommended for production and Drupal/SSR): Set `name` and optionally `sprite-url`. The component renders `<svg><use href="...#name">` — this works server-side without JavaScript and avoids content shift before hydration.

**Inline fetch mode**: Set `src` to a URL of a standalone `.svg` file. The component fetches the file, sanitizes the markup (stripping `<script>` elements, `on*` event-handler attributes, `javascript:` URIs, `<foreignObject>`, and `style` attributes), and embeds the sanitized SVG directly into the shadow root. This mode requires browser JavaScript and is not server-side renderable. For Drupal/Twig use sprite mode instead.

**Use `hx-icon` when:** you need a consistent, accessible, scalable icon that integrates with a token-based design system. Common healthcare use cases include status indicators, action affordances in buttons or menus, navigation items, alert and notification badges, and informational iconography in clinical data displays.

**Do not use `hx-icon`** for complex illustrations or bitmap images — use `<img>` or `hx-image` for those.

## Basic Usage

```html
<!-- Sprite mode (recommended) — no sprite-url needed for inline sprite refs -->
<hx-icon name="check"></hx-icon>

<!-- Sprite mode with explicit sprite sheet URL -->
<hx-icon name="check" sprite-url="/assets/icons/sprite.svg"></hx-icon>

<!-- Informative icon — provides accessible name to screen readers -->
<hx-icon name="check-circle" label="Success: form submitted"></hx-icon>

<!-- Decorative icon — aria-hidden="true" applied automatically -->
<hx-icon name="check"></hx-icon>
<span>Saved successfully</span>

<!-- Size variant -->
<hx-icon name="alert-circle" hx-size="lg" label="Alert"></hx-icon>

<!-- Inline fetch mode — fetches and sanitizes an external SVG file -->
<hx-icon src="/assets/icons/custom-logo.svg" label="Company logo"></hx-icon>

<!-- Color via CSS custom property -->
<hx-icon name="heart" label="Favorite" style="--hx-icon-color: #dc2626;"></hx-icon>

<!-- Color via currentColor inheritance -->
<span style="color: #2563eb;">
  <hx-icon name="info" label="Info"></hx-icon>
  Informational note
</span>
```

## Properties

| Property    | Attribute    | Type                                   | Default     | Description                                                                                                                                                                                                                                                  |
| ----------- | ------------ | -------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `name`      | `name`       | `string`                               | `''`        | Icon name used as the fragment identifier for a sprite sheet reference. Provide the bare symbol id (e.g. `check`). The component builds `${spriteUrl}#${name}`. If `name` already starts with `#` it is used as-is as an inline sprite reference.            |
| `src`       | `src`        | `string \| undefined`                  | `undefined` | URL of a standalone SVG file to fetch and render inline. Takes precedence over sprite mode when both `src` and `name`/`sprite-url` are set. Requires browser JavaScript — use sprite mode in Drupal/SSR contexts.                                            |
| `spriteUrl` | `sprite-url` | `string \| undefined`                  | `undefined` | Base URL of the SVG sprite sheet. Combined with `name` to build the `<use>` href: `${spriteUrl}#${name}`.                                                                                                                                                    |
| `size`      | `hx-size`    | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl'` | `'md'`      | Size variant controlling the width and height of the icon. Maps to design tokens: `xs`=1rem, `sm`=1.25rem, `md`=1.5rem, `lg`=2rem, `xl`=2.5rem. Override with `--hx-icon-size`.                                                                              |
| `label`     | `label`      | `string`                               | `''`        | Accessible label for the icon. When non-empty, `role="img"` and `aria-label` are applied so assistive technology announces the icon. When empty the icon is treated as decorative and `aria-hidden="true"` is applied. Sprite mode also renders a `<title>`. |

**Note on the `hx-size` attribute name:** The `size` JavaScript property maps to the `hx-size` HTML attribute (not `size`) to avoid colliding with the native `<input size>` attribute in Drupal attribute-passthrough scenarios.

## Events

`hx-icon` is a presentational component and does not dispatch any custom events.

## CSS Parts

| Part  | Description                                                                                                                                                                                                |
| ----- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `svg` | In sprite mode: the `<svg>` element itself. In inline fetch mode: the `<span>` wrapper that contains the fetched SVG markup. Both parts share this name for consistent external styling via `::part(svg)`. |

```css
/* Override icon dimensions from outside the shadow DOM */
hx-icon::part(svg) {
  width: 2rem;
  height: 2rem;
}

/* Apply a stroke-based style to sprite-mode icons */
hx-icon::part(svg) {
  fill: none;
  stroke: currentColor;
  stroke-width: 1.5;
}
```

## CSS Custom Properties

| Property          | Default                    | Description                                                                                       |
| ----------------- | -------------------------- | ------------------------------------------------------------------------------------------------- |
| `--hx-icon-size`  | `var(--hx-size-6, 1.5rem)` | Width and height of the icon. Overrides the size variant when set directly on the host.           |
| `--hx-icon-color` | `currentColor`             | Icon fill color. Defaults to `currentColor` so the icon inherits the parent text color naturally. |

```css
/* Fixed size override — takes precedence over hx-size attribute */
hx-icon {
  --hx-icon-size: 2.25rem;
}

/* Brand color for primary icons */
hx-icon.icon--primary {
  --hx-icon-color: var(--hx-color-primary);
}

/* Muted decorative icons */
hx-icon.icon--muted {
  --hx-icon-color: var(--hx-color-neutral-400);
}
```

## Accessibility

`hx-icon` implements the WCAG 2.1 AA pattern for SVG icons: decorative icons are hidden from assistive technology, informative icons carry an accessible name via `role="img"` and `aria-label`.

| Topic             | Details                                                                                                                                                                                                                                                              |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Decorative icons  | When `label` is empty, `aria-hidden="true"` is applied to the SVG/wrapper so screen readers skip it entirely. Use this pattern when adjacent visible text already communicates the icon's meaning.                                                                   |
| Informative icons | When `label` is non-empty, `role="img"` and `aria-label="${label}"` are applied. Sprite mode also injects a `<title>` element inside the SVG for maximum compatibility. Use this pattern for standalone icons that convey meaning without accompanying visible text. |
| Focusability      | `focusable="false"` is set on all SVG elements (inner SVG in inline mode, the outer SVG in sprite mode) to prevent IE11/old-Edge from making SVGs keyboard-focusable.                                                                                                |
| ARIA conflict     | In inline fetch mode, ARIA attributes (`role`, `aria-label`, `aria-hidden`) are stripped from the fetched inner SVG by the sanitizer. The wrapper `<span part="svg">` owns all ARIA semantics, preventing conflicts between inner and outer ARIA state.              |
| XSS sanitization  | Inline fetch mode runs a sanitization pass that removes `<script>`, `<foreignObject>`, `on*` event-handler attributes, `javascript:` and `data:` URIs in URL-bearing attributes, and `style` attributes that could carry CSS injection payloads.                     |
| Keyboard          | `hx-icon` is not interactive and is not keyboard-focusable. When used inside interactive elements (buttons, links), focus and keyboard handling belong to the parent.                                                                                                |
| Color contrast    | Icon color defaults to `currentColor`. Ensure the surrounding context meets WCAG 2.1 AA contrast ratios (3:1 for graphical elements against their background per WCAG 1.4.11).                                                                                       |
| WCAG 2.1 AA       | Verified with axe-core: zero violations in sprite mode (decorative and informative) and inline fetch mode (decorative and informative). All test cases pass.                                                                                                         |

### Choosing Decorative vs. Informative

```html
<!-- CORRECT: Decorative — button text conveys the action -->
<button>
  <hx-icon name="save"></hx-icon>
  Save Changes
</button>

<!-- CORRECT: Informative — icon stands alone, label conveys meaning -->
<hx-icon name="alert-triangle" label="Critical alert: patient allergy on file"></hx-icon>

<!-- INCORRECT: Icon-only button without a label — screen reader gets nothing -->
<button>
  <hx-icon name="trash-2"></hx-icon>
</button>

<!-- CORRECT: Icon-only button — either label the icon or the button -->
<button aria-label="Delete record">
  <hx-icon name="trash-2"></hx-icon>
</button>
```

## Design Tokens

`hx-icon` consumes two component-level CSS custom properties, both of which fall back to semantic tokens.

| Token             | Fallback                   | Notes                                                                        |
| ----------------- | -------------------------- | ---------------------------------------------------------------------------- |
| `--hx-icon-size`  | `var(--hx-size-6, 1.5rem)` | Overridden per size variant via `:host([hx-size="..."])` selectors.          |
| `--hx-icon-color` | `currentColor`             | Inherits parent text color by default. Override to apply a fixed icon color. |

Size variant token map:

| `hx-size` | Token          | Value     |
| --------- | -------------- | --------- |
| `xs`      | `--hx-size-4`  | `1rem`    |
| `sm`      | `--hx-size-5`  | `1.25rem` |
| `md`      | `--hx-size-6`  | `1.5rem`  |
| `lg`      | `--hx-size-8`  | `2rem`    |
| `xl`      | `--hx-size-10` | `2.5rem`  |

## Drupal Integration

Use sprite mode in Twig templates — it works server-side without JavaScript and avoids content shift during hydration. Inline fetch mode requires JavaScript and should be avoided in Drupal page renders.

```twig
{# Decorative icon — no label, aria-hidden applied automatically #}
<hx-icon name="{{ icon_name }}"></hx-icon>

{# Informative icon — label required for standalone icon usage #}
<hx-icon
  name="{{ icon_name }}"
  label="{{ icon_label }}"
></hx-icon>

{# With explicit sprite sheet URL (CDN or static asset path) #}
<hx-icon
  name="{{ icon_name }}"
  sprite-url="{{ base_path ~ directory }}/dist/icons/sprite.svg"
  {% if icon_label %}label="{{ icon_label }}"{% endif %}
></hx-icon>

{# With size variant #}
<hx-icon
  name="{{ icon_name }}"
  hx-size="{{ icon_size|default('md') }}"
  {% if icon_label %}label="{{ icon_label }}"{% endif %}
></hx-icon>
```

Load the library in your module's `.libraries.yml`:

```yaml
helix-components:
  js:
    /libraries/helix/dist/index.js: {}
```

Example from a Drupal alert paragraph template:

```twig
{# paragraph--alert-message.html.twig #}
{% set alert_icons = {
  'success': 'check-circle',
  'warning': 'alert-triangle',
  'error':   'x-circle',
  'info':    'info',
} %}
{% set icon_name = alert_icons[alert_type|default('info')] %}

<div class="alert alert--{{ alert_type }}">
  <hx-icon
    name="{{ icon_name }}"
    hx-size="lg"
    label="{{ alert_type|capitalize }}"
    sprite-url="{{ base_path ~ directory }}/dist/icons/sprite.svg"
  ></hx-icon>
  <div class="alert__body">{{ content.field_body }}</div>
</div>
```
