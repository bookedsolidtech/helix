HELIX offers three enterprise support tiers designed for healthcare organizations at different scales. All tiers include access to the public GitHub issue tracker. Enterprise tiers add guaranteed response times, dedicated communication channels, and proactive architectural guidance.

## Support Tiers

### Community (Free)

Available to all users of the open-source library.

| Feature | Details |
| --- | --- |
| GitHub Issues | Public tracker, best-effort response |
| Documentation | Full access to docs.wc-2026.dev |
| Storybook | Full access to component playground |
| Security advisories | Public CVE notifications |
| Response time | No guarantee |

### Professional

For teams shipping HELIX-based products in production.

| Feature | Details |
| --- | --- |
| Everything in Community | Included |
| Dedicated Slack channel | Private channel with HELIX engineers |
| Response time — critical bugs | 4 business hours |
| Response time — questions | 2 business days |
| Minor version upgrade assistance | Included |
| Seats | Up to 10 team members |
| Annual cost | Contact sales |

### Enterprise

For healthcare organizations with compliance requirements and multi-team deployments.

## Dedicated Channel Access

Enterprise and Professional customers receive a private Slack channel in the HELIX workspace. The channel includes:

- Direct access to HELIX core engineers
- Advance notice of breaking changes and deprecations
- Pre-release access to major versions for integration testing
- Escalation path for time-sensitive issues

## Quarterly Architecture Reviews

Enterprise customers are entitled to quarterly 60-minute architecture review sessions. Sessions typically cover:

- Current integration patterns and potential improvements
- Upcoming HELIX roadmap items relevant to your stack
- Design token structure and theming strategy
- Accessibility audit findings and remediation planning
- Performance profiling and bundle size optimization

Sessions are conducted via video conference. Request a session through your dedicated account engineer.

## Escalation Path

| Priority | Definition | Initial Response | Updates |
| --- | --- | --- | --- |
| P1 — Critical | Production system down or data risk | 1 hour (24/7) | Every 2 hours |
| P2 — High | Major feature broken, workaround unavailable | 4 hours | Every 4 hours |
| P3 — Medium | Feature degraded, workaround available | 1 business day | Every 2 business days |
| P4 — Low | Question, enhancement, or documentation issue | 2 business days | Weekly |

For P1 issues, use the emergency contact number provided in your Enterprise onboarding package. Do not rely solely on Slack for critical escalations.
